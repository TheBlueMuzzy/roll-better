     - joining mode: shows "CREATE" but dimmed/inactive appearance. Tapping cancels join (clear code chars, set mode to 'idle', then immediately start create flow)
   - JOIN button:
     - idle mode: shows "JOIN" (disabled until code complete... actually just shows "JOIN")
     - creating mode: shows "JOIN". Tapping cancels create (room.leave(), reset state, set mode to 'joining', show code inputs)
     - joining mode: shows "START" when code complete (calls room.joinRoom then the join completes via useRoom). Before code complete, shows "JOIN" disabled.
4. Conditional inline section:
   - creating mode: Room code "XXXX Tap to Copy" button + "Share this code with friends" text + player list (color dot, name, star for host, ready checkmark). No "total players" line.
   - joining mode: 4 code input boxes (reuse exact logic from LobbyScreen: handleCodeInput, handleCodeKeyDown, handleCodePaste, refs). Error display below.
5. Ready toggle (only visible when connected as non-host in creating mode... actually non-hosts would be in a "joined" state. Wait — if you JOIN someone else's room, you're not host. But in THIS MainMenu, you can only be host if you created. If you joined, you see the player list too but with a READY toggle instead of START.)

**Wait — re-thinking the join flow:**
When user taps JOIN and enters a code, the Join button becomes START. Tapping START calls room.joinRoom(code, randomName, color). Once connected (room_state received), the UI should show the player list (same as create mode) but with a READY toggle instead of START. The mode transitions: joining → 'joined' (new state).

Actually, let me simplify. The state machine should be:
- `idle`: no online flow active
- `creating`: user pressed Create, room is being created/connected, shows room code + player list. Create button = START.
- `joining`: user pressed Join, showing code inputs. Join button = START (when code complete).
- `joined`: user successfully joined someone else's room. Shows player list with READY toggle. No START button (host starts).

**Transitions:**
- idle → creating: tap Create
- idle → joining: tap Join
- creating → joining: tap Join (calls room.leave(), clears state)
- creating → game: host taps START → game_starting fires
- joining → joined: room.joinRoom succeeds (isConnected && roomCode && !isHost)
- joining → creating: tap Create (clears code, starts create)
- joined → idle: tap Create or Join (calls room.leave(), resets)
- joined → game: host starts → game_starting fires

**Game start detection:** Same useEffect as LobbyScreen — watch room.gameStartData and call onGameStart.

**Remove:** Player count selector (PLAYER_COUNTS array, playerCount state, selector UI). Name input. "PLAY ONLINE" button. menu-online class usage.

**How to Play:** Replace the text link in menu-links with a ? icon button positioned absolute bottom-left (mirror of gear icon). Use class `menu-help`.

**Upgrades:** Keep in menu-links (now just the single Upgrades button).

**Move from LobbyScreen:** SILLY_NAMES array, getRandomName function, code input state/handlers, clipboard copy logic, error display.

**types/game.ts:** Remove 'lobby' from the screen union type: `screen: 'menu' | 'game' | 'winners'`
