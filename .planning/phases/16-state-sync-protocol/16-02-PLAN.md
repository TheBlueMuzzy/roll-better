<task type="auto">
  <name>Task 1: Unlock and skip handlers</name>
  <files>party/server.ts</files>
  <action>
Implement `handleUnlockRequest` and `handleSkipUnlock` in the server.

**Unlock collection pattern (wait-for-all):**
The server must wait for ALL online players to either unlock or skip before advancing. Use the `unlockResponses` Map on gameState to track responses.

1. `handleUnlockRequest(sender, slotIndices)`:
   - Validate: game exists, phase is 'unlocking', sender is an online player in the game
   - Validate slot indices: each must be a currently locked slot belonging to this player
   - Store response: `unlockResponses.set(senderId, { type: 'unlock', slotIndices })`
   - Check if all online players have responded → if yes, call `processAllUnlocks()`

2. `handleSkipUnlock(sender)`:
   - Validate: game exists, phase is 'unlocking', sender is online player
   - Validate: player must have poolSize > 0 OR lockedDice.length >= 8 (can't skip if poolSize=0 and not complete)
   - Store response: `unlockResponses.set(senderId, { type: 'skip' })`
   - Check if all online players have responded → if yes, call `processAllUnlocks()`

3. `processAllUnlocks()`:
   - Process AI bot unlock decisions first:
     - Import `getAIUnlockDecision` from `../src/utils/aiDecision`
     - For each bot player: compute unlock decision, apply to server state
   - Process human player unlock decisions from `unlockResponses`:
     - For 'unlock': remove locked dice at selected slots, poolSize += 2 * unlockCount
     - For 'skip': no state change
   - Broadcast `unlock_result` for each player who unlocked (so clients can animate)
   - Clear `unlockResponses`
   - Set phase to 'idle', broadcast `phase_change: idle`

**Edge case:** Must-unlock guard — if a player has poolSize === 0 and lockedDice.length < 8, they MUST unlock (can't skip). Server should reject skip_unlock in this case with an error message.

Add `"unlock_request"` and `"skip_unlock"` cases to the onMessage switch.

<task type="auto">
  <name>Task 2: Scoring, handicap, and round transitions</name>
  <files>party/server.ts</files>
  <action>
Implement the end-of-round pipeline: winner detection → scoring → handicap → next round (or session end).

**Wire into the locking→unlocking transition (from Plan 16-01 Task 3):**
The 1-second timer after locking should call a new `checkWinnerOrUnlock()` method:

1. `checkWinnerOrUnlock()`:
   - Check if any player has lockedDice.length === 8
   - If YES → call `handleScoring(winnerId)`
   - If NO → set phase to 'unlocking', broadcast `phase_change: unlocking`, clear unlockResponses

2. `handleScoring(winnerId)`:
   - Compute round score using same formula as gameStore.ts:
     - Winner's poolSize = remaining dice
     - Penalties: [1, 0, 1, 1] for poolSize 1-4
     - roundScore = max(0, 8 - sum of applicable penalties)
   - Add roundScore to winner's score in server state
   - Set phase to 'scoring'
   - Broadcast `scoring` message with winnerId, roundScore, and all player states
   - After 2-second timeout → call `handleHandicapAndNextRound()`

3. `handleHandicapAndNextRound()`:
   - Apply handicap to all players:
     - Winner: startingDice = max(1, startingDice - 1)
     - Others: startingDice = min(12, startingDice + 1)
   - Check session end: any player score >= 20
   - If session over → set phase to 'sessionEnd', broadcast `session_end` with final player states
   - If not over → set phase to 'roundEnd', broadcast `phase_change: roundEnd`
   - After 500ms → call `serverInitRound()` (generates new goals, resets pools, broadcasts round_start)

**Timer management:** Store timer IDs as class properties so they can be cleared if the room closes mid-game. Add cleanup in `onClose` or a new `cleanupTimers()` method.

**Scoring formula reference (from gameStore.ts):**
```
const penalties = [1, 0, 1, 1];
let penalty = 0;
for (let i = 0; i < poolSize && i < penalties.length; i++) {
  penalty += penalties[i];
}
roundScore = Math.max(0, 8 - penalty);
```
