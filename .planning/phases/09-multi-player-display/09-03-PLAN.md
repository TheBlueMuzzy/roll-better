**Established patterns:**
- AnimatingDie: fromPos → toPos with ease-in-out cubic, parabolic Y arc, slerp rotation
- Human lock animations: computed in setRollResults, stored as lockAnimations in roundState
- Scene renders AnimatingDie instances outside Physics group, tracks completion via refs
- animatingSlotIndices hides PlayerRow dice during flight

**Current AI lock behavior:**
- AI auto-locks are computed in setRollResults alongside human locks (gameStore.ts)
- AI locks are instant — lockedDice updated immediately, no animation data generated
- AI PlayerRows in Scene.tsx just read lockedDice and render static dice

**Design direction (from CONTEXT.md):**
- AI lock: dice lerp FROM the AI profile group TO their locked row slots
- Start at scale 0, scale up along the way to full size at destination
- "Dice emerging from the AI's pool into locked position"

<task type="auto">
  <name>Task 1: Extend AnimatingDie with scale interpolation and create AI lock animation data</name>
  <files>src/components/AnimatingDie.tsx, src/types/game.ts, src/store/gameStore.ts</files>
  <action>
**AnimatingDie changes:**
- Add optional `fromScale` and `toScale` props (default both to 1 for backward compatibility).
- In the useFrame loop, interpolate scale: `currentScale = fromScale + (toScale - fromScale) * eased`.
- Apply scale to the mesh group: `groupRef.current.scale.setScalar(currentScale)`.
- When fromScale=0, toScale=1: die grows from nothing. This is the AI lock animation.
- Existing human lock animations don't pass these props, so they still animate at scale 1 throughout.

**AI lock animation data (gameStore.ts → setRollResults):**
- Currently AI locks are instant. Change: compute AI lock animation data alongside human lock animation data.
- For each AI player's new locks, create LockAnimation entries:
  - `fromPos`: the AI player's profile group position (needs to be derivable — use a convention like: X = leftmost slot X - profileOffset, Y = DIE_SIZE/2, Z = player row Z). Store a helper or pass profile positions.
  - `toPos`: the target slot position in the AI player's row (getSlotX(slotIndex), DIE_SIZE/2, 0 — but relative to the AI row's Z, so absolute Z = playerRowZ).
  - `fromRotation`: [0, 0, 0] (starting from profile, no initial rotation needed).
  - `value`: the locked die face value.
  - `delay`: stagger like human locks (0.25-0.5s spacing).
  - `fromScale`: 0, `toScale`: 1 (new fields for scale interpolation).

**Store changes:**
- Add `aiLockAnimations: LockAnimation[]` to RoundState (separate from human lockAnimations to avoid confusion).
- Or better: add a `playerId` field to LockAnimation and store all lock animations together. Scene can filter by player.
- Add `fromScale`/`toScale` optional fields to LockAnimation interface.
- Compute AI lock animations in setRollResults where AI auto-locks already happen.
- Add `clearAILockAnimations` action (or extend clearLockAnimations to clear all).

**LockAnimation interface update:**
Add optional fields: `fromScale?: number`, `toScale?: number`, `playerId?: string`. Existing human animations don't set these (defaults to scale 1, playerId = player-0).

<task type="auto">
  <name>Task 2: Render AI lock animations in Scene and hide animating slots</name>
  <files>src/components/Scene.tsx, src/components/PlayerRow.tsx</files>
  <action>
**Scene.tsx changes:**
- Read AI lock animations from store (aiLockAnimations or filtered by playerId).
- Render AnimatingDie instances for each AI lock animation, outside the Physics group (same pattern as human lock animations).
- Pass fromScale={0} toScale={1} to these AnimatingDie instances.
- Track AI lock animation completion separately (or combine with human tracking).
- On all AI lock animations complete, clear them from store.
- The AI lock animations should play SIMULTANEOUSLY with human lock animations (both triggered by setRollResults in the same roll cycle). No need to sequence them — all players lock at the same time in simultaneous play.

**PlayerRow changes for AI rows:**
- AI PlayerRows need to know which slots are currently animating in (to hide them, same as human's animatingSlotIndices).
- Pass per-player animatingSlotIndices to each AI's PlayerRow.
- Compute these from the AI lock animations: for each AI player, extract the goalSlotIndex values of their active lock animations.

**Position computation:**
- AI lock animations need absolute world positions for fromPos and toPos.
- fromPos: AI profile group position = [profileX, 0, aiRowZ] where profileX is left of the row and aiRowZ = -3.77 + (aiIndex + 1) * 0.9.
- toPos: [getSlotX(slotIndex) + rowOffsetX, DIE_SIZE/2, aiRowZ] where rowOffsetX is the same shift applied in 09-01.
- These positions need to be in world space since AnimatingDie renders at scene root level.

Ensure the die color matches the AI player's color (already available from players array).
