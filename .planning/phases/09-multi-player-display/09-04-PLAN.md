**Established patterns:**
- Human unlock: MitosisDie (3-phase: lerp → shake → split) — complex, communicates "1 die → 2"
- AnimatingDie with fromScale/toScale (added in 09-03) — supports scale interpolation
- processAIUnlocks in gameStore: currently instant state update, called after human unlock completes
- AI unlock flow: human confirms/skips → human animations play → processAIUnlocks → phase idle

**Current AI unlock behavior:**
- processAIUnlocks() applies AI unlock decisions instantly (no animation)
- Called in App.tsx after human unlock animations complete (or on skip)
- Each AI player's getAIUnlockDecision() returns slots to unlock

**Design direction (from CONTEXT.md):**
- Reverse of lock: dice scale DOWN as they fly in an arc back to the AI profile group
- When they reach scale 0, they disappear (returned to data/pool)
- Simpler than human mitosis — no shake, no split. Just a clean return-to-avatar.

  <name>Task 1: Create AI unlock animation data and store actions</name>
  <files>src/types/game.ts, src/store/gameStore.ts, src/App.tsx</files>
  <action>
**New type — AIUnlockAnimation:**
- `playerId`: string (which AI player)
- `slotIndex`: number (which goal slot being unlocked)
- `value`: number (face value of the die)
- `fromPos`: [number, number, number] (absolute world position of the slot in AI's row)
- `toPos`: [number, number, number] (AI profile group position — same as lock animation's fromPos)
- `delay`: number (stagger timing)

Add to RoundState: `aiUnlockAnimations: AIUnlockAnimation[]`
Add store actions: `setAIUnlockAnimations`, `clearAIUnlockAnimations`

**Refactor processAIUnlocks flow (App.tsx):**
Currently processAIUnlocks is called and instantly applies state. Change to a two-step flow:

1. **Compute animations**: For each AI player, call getAIUnlockDecision to get slots to unlock. Build AIUnlockAnimation entries:
   - fromPos: [getSlotX(slotIndex) + rowOffsetX, DIE_SIZE/2, aiRowZ] (world position of the locked slot)
   - toPos: [profileX, 0, aiRowZ] (AI profile group position)
   - delay: stagger 0.15-0.3s between each die
2. **Store animations**: setAIUnlockAnimations(animations)
3. **Wait for animations to complete**: setTimeout based on last animation's delay + ~0.6s duration
4. **Apply state changes**: After animations finish, call processAIUnlocks() to actually update lockedDice/poolSize
5. **Clear animations**: clearAIUnlockAnimations()
6. **Proceed to idle**: setPhase('idle')

This replaces the current instant processAIUnlocks call. The state changes happen AFTER the animations play, so the dice visually fly away before disappearing from state.

Handle the case where AI has no unlocks — skip animation, go straight to state update.

<task type="auto">
  <name>Task 2: Render AI unlock animations in Scene</name>
  <files>src/components/Scene.tsx, src/components/PlayerRow.tsx</files>
  <action>
**Scene.tsx:**
- Read aiUnlockAnimations from store.
- For each AIUnlockAnimation, render an AnimatingDie with:
  - fromPos = animation.fromPos (slot position in AI row)
  - toPos = animation.toPos (AI profile group position)
  - fromScale = 1, toScale = 0 (shrink to nothing)
  - value = animation.value
  - color = AI player's color
  - delay = animation.delay
  - duration = ~0.5s (faster than human mitosis — clean, simple departure)
  - onComplete callback to track completion
- Render outside Physics group (same as all animation components).
- Track completion: when all AI unlock animations complete, fire the callback that applies state and clears animations.

**PlayerRow changes:**
- AI PlayerRows need to hide slots that are currently animating out (similar to how animatingSlotIndices works for locks).
- Compute "unlocking slot indices" from aiUnlockAnimations for each AI player.
- Pass these to the AI PlayerRow so it hides those slots during the fly-out animation.
- After animations complete and state is applied, the slots are naturally empty.

**Timing integration:**
- AI unlock animations should start AFTER human unlock animations finish (or after human skips).
- Current flow: human animates → processAIUnlocks → idle.
- New flow: human animates → AI unlock animations start → AI animations complete → processAIUnlocks (state) → idle.
- If human skips unlock: AI unlock animations start immediately → complete → state → idle.
