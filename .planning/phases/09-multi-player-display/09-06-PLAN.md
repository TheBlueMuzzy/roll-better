- Each player has lockedDice: LockedDie[] with { goalSlotIndex, value }

- players[].lockedDice[].goalSlotIndex tells which goal slot a player has locked
- To show "who leads on column X": check which players have goalSlotIndex === X in their lockedDice

- Renders as a 3D group positioned just ABOVE the goal row (Z slightly less than goal Z, or Y offset to sit just under the dice visually).

  - Render a small colored circle (or dot) below/behind the goal die at that slot.
  - Use a small Three.js circle mesh (CircleGeometry, radius ~0.12) or a drei Html dot.

**Single player on a slot:**
- Render a solid circle in that player's color.

<task type="auto">
  <name>Task 2: Handle ties — split circles when multiple players share a slot</name>
  <files>src/components/GoalIndicators.tsx</files>
  <action>
**Tie handling — multiple players locked on the same goal slot:**

When 2+ players have the same goalSlotIndex locked, render a split indicator:

**2-player tie:**
- Two half-circles (semicircles), each in one player's color.
- Use two CircleGeometry meshes with `thetaStart` and `thetaLength` to create semicircles:
  - Player A: thetaStart = 0, thetaLength = PI (top half)
  - Player B: thetaStart = PI, thetaLength = PI (bottom half)
- Same position as single dot, same radius.

**3+ player tie:**
- Divide the circle into equal wedges: thetaLength = 2*PI / playerCount.
- Each wedge: thetaStart = i * (2*PI / playerCount), thetaLength = (2*PI / playerCount).
- Each wedge in the corresponding player's color.

**Implementation:**
- In the render loop for each slot, count how many players have it locked.
- If 1: single full circle.
- If 2+: N wedge meshes, each with computed theta values.
- CircleGeometry(radius, segments, thetaStart, thetaLength) supports this natively.

**Visual polish:**
- Add a thin dark outline ring around each indicator (slightly larger CircleGeometry with dark material behind the colored wedges) for visibility against the dark wood surface.
- Or use a RingGeometry for the outline.

**Reactivity:**
- GoalIndicators should update whenever any player's lockedDice changes.
- Since it reads from the store via props (or direct useGameStore selector), it will re-render on lock changes.
- Indicators should appear/disappear as locks happen — they update in real-time during locking phase.
