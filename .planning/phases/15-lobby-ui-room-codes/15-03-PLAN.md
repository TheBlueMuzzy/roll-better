<task type="auto">
  <name>Task 1: Build LobbyScreen component with create/join and lobby views</name>
  <files>src/components/LobbyScreen.tsx, src/App.tsx</files>
  <action>
Create LobbyScreen.tsx with two internal views managed by local state:

**Props:** `{ visible: boolean; onGameStart: (players: RoomPlayer[], targetPlayers: number, aiDifficulty: string) => void; onBack: () => void }`

**CreateJoinView** (shown when not connected):
- Name input: text field, max 12 chars, placeholder "Your name". Store in local state. Required before create/join.
- "CREATE ROOM" button: calls `useRoom().createRoom(name, color)`. Color = PLAYER_COLORS[0] (host always gets first color). Import PLAYER_COLORS from gameStore (it's already exported at module level — if not, the plan executor should export it).
- Divider: "— or join a room —"
- Room code input: 4 individual `<input>` elements, each maxLength=1, inputMode="text", autoCapitalize="characters". On input: uppercase the value, auto-advance focus to next input. On backspace in empty input: focus previous. On paste: distribute chars across all 4 inputs. Store code as single string in state.
- "JOIN" button: enabled when code is 4 letters. Calls `useRoom().joinRoom(code, name, color)`. Color = assigned after joining based on player count in room (will be corrected by room_state).
- "BACK" button: calls onBack prop.
- Error display: show useRoom().error below buttons if present (red text, auto-fades).

**LobbyView** (shown when connected — isConnected && roomCode):
- Room code display: large text (2rem+), letter-spaced, with "TAP TO COPY" label. On click, copy to clipboard via navigator.clipboard.writeText(). Brief "Copied!" feedback.
- "Share this code with friends" subtitle.
- Player list: map useRoom().players to list items. Each item shows:
  - Colored circle (player.color, 24px)
  - Name (bold if isHost, "★" prefix if isHost)
  - Ready indicator: green checkmark if isReady, gray dash if not
  - "(You)" suffix if player.id === playerId
- Ready toggle: button at bottom, text "READY" / "NOT READY" (toggles). Calls useRoom().toggleReady(). Styled: green when ready, outline when not.
- Host controls (only visible when isHost):
  - Player count target: selector buttons 2-8 (same pattern as MainMenu player count). Default = current online player count or 4, whichever is larger. This determines total players including AI fill.
  - AI difficulty: selector buttons Easy/Medium/Hard (same pattern as MainMenu).
  - "START GAME" button: enabled when all non-host players are ready (or solo). Calls useRoom().startGame(targetPlayers, difficulty).
- "LEAVE" button: calls useRoom().leave() then onBack().

**Game start detection:** useEffect watching useRoom().gameStartData. When non-null, call onGameStart(gameStartData.players, gameStartData.targetPlayers, gameStartData.aiDifficulty).

**Fade transition:** Use the same mount → rAF → add class pattern as MainMenu (showClass state).

**Wire in App.tsx:** Replace the lobby placeholder div with the actual LobbyScreen component. Pass onGameStart and onBack handlers.
