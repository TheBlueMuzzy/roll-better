  <name>Task 1: Create useRoom React hook for lobby connection management</name>
  <files>src/hooks/useRoom.ts</files>
  <action>
Create a new hook that wraps partyClient.ts utilities in React-friendly state management.

**Exported interface:**
```
useRoom() returns {
  // State (reactive)
  roomCode: string | null,
  playerId: string | null,
  players: RoomPlayer[],
  hostId: string | null,
  isConnected: boolean,
  isHost: boolean,
  status: RoomStatus | null,
  error: string | null,
  gameStartData: { players: RoomPlayer[], targetPlayers: number, aiDifficulty: string } | null,

  // Actions
  createRoom: (playerName: string, color: string) => void,
  joinRoom: (code: string, playerName: string, color: string) => void,
  leave: () => void,
  toggleReady: () => void,
  startGame: (targetPlayers: number, aiDifficulty: string) => void,
}
```

**Room code generation** (used by createRoom):
Generate a random 4-letter uppercase code: pick 4 chars from "ABCDEFGHJKLMNPQRSTUVWXYZ" (exclude I and O to avoid confusion with 1 and 0). Use `Math.random()` — no crypto needed for room codes.

**Connection lifecycle:**
- `createRoom`: generate code → `createPartyConnection(code)` → attach listeners → on "connected" message, send "join" with name/color
- `joinRoom`: validate code is 4 uppercase letters → `createPartyConnection(code)` → attach listeners → on "connected", send "join"
- `leave`: send "leave" message → close socket → reset all state
- Socket `onmessage`: parse with `parseServerMessage()`, update state based on message type:
  - "connected": store playerId, roomCode
  - "room_state": update players, hostId, status. Derive isHost from hostId === playerId.
  - "player_joined": append to players array
  - "player_left": filter player from array
  - "game_starting": store in gameStartData (App.tsx will consume this to start the game)
  - "error": set error state, auto-clear after 3 seconds
- Socket `onclose`: reset connection state, set error "Connection lost"

**Cleanup:** On unmount, close socket if open. Use useRef for the socket (not state — avoids re-renders on socket object changes). Use useState for all reactive state.

**Important:** Do NOT import from gameStore. This hook is purely for room/connection state. The App.tsx layer bridges between useRoom and gameStore.
