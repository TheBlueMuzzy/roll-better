  <name>Task 1: Extend protocol types with ready, start_game, and game_starting messages</name>
  <files>src/types/protocol.ts</files>
  <action>
Add three new message types to the existing discriminated unions:

**Client → Server (add to ClientMessage union):**
- `ReadyMessage { type: "ready" }` — toggles the sender's ready state
- `StartGameMessage { type: "start_game"; targetPlayers: number; aiDifficulty: string }` — host requests game start. `targetPlayers` is total desired players (2-8), `aiDifficulty` is "easy"|"medium"|"hard". Use `string` for aiDifficulty (not the game.ts AIDifficulty type — protocol.ts must not import game types).

**Server → Client (add to ServerMessage union):**
- `GameStartingMessage { type: "game_starting"; players: RoomPlayer[]; targetPlayers: number; aiDifficulty: string }` — broadcast to all clients when host starts the game. Contains the final online player list so each client can set up locally.

Keep the existing messages untouched. Follow the established pattern: each message is its own interface, added to the union type.

<task type="auto">
  <name>Task 2: Update server to handle ready toggle and host-only game start</name>
  <files>party/server.ts</files>
  <action>
Add two new cases to the `onMessage` switch statement:

**"ready" handler:**
1. Guard: sender must be in players map (ignore if not joined yet)
2. Toggle `player.isReady` (flip boolean)
3. Broadcast updated `room_state` to all players (existing `broadcastRoomState()` helper)

**"start_game" handler:**
1. Guard: sender must be host (`sender.id === this.hostId`). If not, send error "Only the host can start the game".
2. Guard: room must have at least 1 player (the host). If 0 others, that's OK — AI will fill on client side.
3. Guard: all non-host players must have `isReady === true`. If any aren't ready, send error "All players must be ready".
4. Set `this.status = "playing"` (prevents new joins)
5. Extract `targetPlayers` and `aiDifficulty` from the parsed message
6. Broadcast `game_starting` message with: `players` (Array.from this.players.values()), `targetPlayers`, `aiDifficulty`
7. Log: `Game starting: ${this.players.size} online, ${targetPlayers} target`

Import the new message types (StartGameMessage) at the top alongside existing imports. The switch cases use the `parsed.type` discriminant — add `"ready"` and `"start_game"` cases.

Do NOT modify existing handlers (join, leave). Do NOT change the onConnect/onClose logic.
  </action>
