**src/store/gameStore.ts — add initOnlineGame action:**

Add a new action to the store interface and implementation:

```
initOnlineGame: (onlinePlayers: { id: string; name: string; color: string }[], localPlayerId: string, targetPlayers: number, aiDifficulty: AIDifficulty) => void
```

Implementation:
1. Find the local player in onlinePlayers by localPlayerId. If not found, use first player.
2. Build the players array:
   a. Index 0 = local player: `{ id: localPlayerId, name: localPlayer.name, color: localPlayer.color, isAI: false, difficulty: undefined, ... }` (same defaults as initGame: score 0, startingDice 2, poolSize 2, empty locked/selected)
   b. Indices 1+ = other online players (in join order, excluding local): `{ id: p.id, name: p.name, color: p.color, isAI: true, difficulty: aiDifficulty, ... }`. Mark them `isAI: true` because in Phase 15 they play locally as AI. Phase 16 will change this to sync.
   c. If total online players < targetPlayers, fill remaining slots with AI: `{ id: "ai-N", name: "Bot N", color: PLAYER_COLORS[index], isAI: true, difficulty: aiDifficulty }`. Use colors not already taken by online players.
3. Set state: `{ players, phase: 'lobby', currentRound: 0, roundState: initialRoundState, shownTips: [] }` — same pattern as initGame.

**Lobby → game transition cleanup:**
- When transitioning from lobby to game (handleOnlineGameStart), the room connection should be kept alive (Phase 16 will use it for sync). For now, do NOT disconnect. The useRoom hook stays mounted but inactive during game.
- If useRoom detects the room is no longer needed (Phase 15 doesn't need sync), it's fine to leave the connection open — it will close when the component unmounts or user returns to menu.

**Return to menu from game:**
- Update handleMenu in App.tsx: if a room connection exists, call leave() on useRoom before returning to menu. Since useRoom is inside LobbyScreen which unmounts when screen !== 'lobby', this happens automatically on unmount cleanup. Verify this is the case.
- If LobbyScreen stays mounted during game (for future Phase 16 sync), add explicit cleanup.

**LobbyScreen onGameStart prop update:**
- Update the prop to pass localPlayerId: `onGameStart: (players: RoomPlayer[], targetPlayers: number, aiDifficulty: string, localPlayerId: string) => void`
- In the useEffect that watches gameStartData, include playerId from useRoom.

**Error handling in lobby:**
- If connection drops while in lobby (onclose fires), show error "Connection lost" and allow retry or return to create/join view.
- If room is closed/full on join attempt, show the server's error message.

**Back from lobby:**
- The onBack callback should call useRoom().leave() if connected, then setScreen('menu').

**Edge case — host leaves:**
- If host disconnects, server migrates host. The new host's client gets room_state with updated hostId. useRoom updates isHost automatically. LobbyView should reactively show host controls to the new host.
