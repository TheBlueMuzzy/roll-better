**Create `src/utils/gatherPoints.ts`** — pure utility, no React dependencies.

```typescript
/**
 * Generate N positions arranged in a circle around a center point on the floor (Y=0).
 * Returns array of [x, y, z] tuples.
 *
 * @param center - [x, y, z] touch position on floor
 * @param count - number of goal points (= pool size)
 * @param radius - circle radius (default: scaled by count, min 1.0, max 3.0)
 * @param rotationOffset - radians to rotate the whole ring (for future orbit animation in Phase 41)
 */
export function getGatherPoints(
  center: [number, number, number],
  count: number,
  radius?: number,
  rotationOffset?: number
): [number, number, number][]
```

**Logic:**
- If count === 0 or center is null, return empty array
- Default radius: `Math.min(3.0, Math.max(1.0, 0.5 + count * 0.25))` — grows with die count but capped
- Y coordinate: `DIE_SIZE / 2` (half die height — dice will float to these points in Phase 41)
- For each i in 0..count-1:
  - angle = rotationOffset + (i / count) * 2π
  - x = center[0] + radius * cos(angle)
  - z = center[2] + radius * sin(angle)
- Clamp x to [ROLLING_X_OFFSET - ARENA_HALF_X + 0.5, ROLLING_X_OFFSET + ARENA_HALF_X - 0.5] to keep points inside walls
- Clamp z to [ROLLING_Z_MIN + 0.5, ROLLING_Z_MAX - 0.5]
- Import ROLLING_X_OFFSET, ARENA_HALF_X, ROLLING_Z_MIN, ROLLING_Z_MAX, DIE_SIZE from RollingArea

**Export the radius calculation separately** so Phase 41 can use the same radius for orbit:
```typescript
export function getGatherRadius(count: number): number
```

This is a pure function — no state, no side effects. Phase 41 will call it per-frame with an incrementing rotationOffset to create orbital motion.
