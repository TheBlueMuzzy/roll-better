**Create `src/components/GatherVisuals.tsx`:**

A single R3F component that reads gatherState from the store and renders all gather visuals. Uses useFrame for smooth animation — no React state for positions.

**Goal point indicators:**
- Call `getGatherPoints(touchPosition, dieCount)` to get positions
- Render a small mesh at each position:
  - Option A (recommended): `<mesh>` with `<sphereGeometry args={[0.15, 8, 8]} />` and emissive material
  - Color: player's color with emissive glow (emissive={playerColor}, emissiveIntensity=2, opacity=0.6, transparent=true)
  - Y position: DIE_SIZE/2 (floor level, half die height)
- Use `useFrame` to smoothly interpolate positions when finger moves (lerp current → target at ~10/s)
- Use refs for mesh positions — update in useFrame via `mesh.position.set(x, y, z)`, NOT via React props

**Vacuum VFX circle:**
- A `<mesh>` with `<ringGeometry>` centered on touchPosition, lying flat on the floor (rotation [-π/2, 0, 0])
- Inner radius: 0 (or very small)
- Outer radius: `getGatherRadius(dieCount) + 0.3` — slightly larger than the goal point circle
- Material: `<meshBasicMaterial>` with player color, transparent, opacity pulsing between 0.15 and 0.35 (use useFrame with sin wave)
- Position: touchPosition[0], 0.02, touchPosition[2] (just above floor to avoid z-fighting)
- Pulse animation: modulate opacity via material ref in useFrame

**Appear/disappear:**
- When gatherState.active transitions to true: visuals appear (can be instant or quick fade-in)
- When active transitions to false: visuals disappear instantly (Phase 42's release will add a burst effect later)

**Wire into Scene.tsx:**
- Import GatherVisuals
- Render `<GatherVisuals />` inside the main `<group>`, OUTSIDE the `<Physics>` wrapper (these are visual-only, no physics interaction)
- Place after the divider mesh, before the Physics group
- GatherVisuals reads its own state from the store (no props needed)

**Performance considerations:**
- The component only renders meshes when gatherState.active is true (early return null when inactive)
- Goal point meshes: use instanced rendering if count > 6, otherwise individual meshes are fine for ≤12 dice
- All animation via useFrame + refs, zero React re-renders during gathering
