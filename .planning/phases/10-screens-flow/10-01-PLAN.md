<task type="auto">
  <name>Task 2: Wire menu into App.tsx — app starts at menu</name>
  <files>src/App.tsx, src/components/Settings.tsx</files>
  <action>
1. In App.tsx:
   - Import MainMenu component
   - Read `screen` and `setScreen` from store
   - REMOVE the auto-start useEffect (lines 59-74 that call initGame on mount). The app should start with screen='menu', showing the MainMenu.
   - Add MainMenu render: when screen === 'menu', render `<MainMenu onPlay={handlePlay} onOpenSettings={() => setSettingsOpen(true)} />`
   - Add handlePlay callback:
     ```
     const handlePlay = useCallback((playerCount: number, difficulty: AIDifficulty) => {
       initGame(playerCount, difficulty);
       initRound();
       setScreen('game');
       // Start pool spawn animation
       const state = useGameStore.getState();
       const humanPlayer = state.players[0];
       if (humanPlayer && humanPlayer.poolSize > 0) {
         const spawnPositions = getSpawnPositions(humanPlayer.poolSize);
         setPoolSpawning(true, spawnPositions);
         const spawnDuration = 600 + humanPlayer.poolSize * 80 + 100;
         setTimeout(() => setPoolSpawning(false), spawnDuration);
       }
     }, [initGame, initRound, setScreen, setPoolSpawning]);
     ```
   - Conditionally render game content: only render Canvas + HUD + TipBanner when screen !== 'menu'. The 3D scene should NOT render while on the menu (saves GPU).
   - Settings overlay should work on BOTH menu and game screens (it's already a separate overlay).

2. In Settings.tsx:
   - Change handleQuitYes to go to menu instead of restarting:
     ```
     const handleQuitYes = () => {
       setQuitConfirm(false);
       onClose();
       setScreen('menu');
     };
     ```
   - Import setScreen from store. Add `const setScreen = useGameStore((s) => s.setScreen);`
   - Change button text from "Quit Game" to "Main Menu"

3. Verify the sessionEnd useEffect still works — it just sets phase to 'sessionEnd', it doesn't need to know about screen state. The winners screen (Plan 10-02) will handle transitioning from sessionEnd.
