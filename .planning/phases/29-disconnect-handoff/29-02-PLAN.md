  <action>
  Create a simple fullscreen overlay that appears when the player disconnects during an active game:

  1. **Create `DisconnectOverlay.tsx`**: A semi-transparent dark overlay (z-index above game, below modals) with centered text:
     - "Connection lost"
     - "Reconnecting..." with a simple animated dots indicator (CSS animation, no library)
     - Style consistent with existing overlays (dark bg, white text, rounded container)

  2. **Show/hide logic**: The `onlineDisconnected` state already exists in useRoom.ts (set true on close during active game, false on reconnect). Use this in App.tsx to conditionally render `<DisconnectOverlay />` when `onlineDisconnected === true`.

  3. **No countdown needed**: The grace window is server-side and the player doesn't know its duration. Just show "Reconnecting..." — PartySocket handles auto-reconnect attempts. If reconnect succeeds, the overlay disappears. If grace expires and seat becomes bot, the existing bot-takeover logic returns them to menu.

  4. **Don't block input**: The overlay is purely visual feedback. The game underneath is paused from the player's perspective (they're offline, no messages flowing). No need for pointer-events:none — the overlay covers everything.
  </action>
  <verify>`npx vite build` succeeds. DisconnectOverlay component exists and is rendered conditionally in App.tsx.</verify>
