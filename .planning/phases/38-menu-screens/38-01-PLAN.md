<objective>
Adjust MainMenu layout and buttons for landscape 16:9.

Purpose: The MainMenu stacks everything vertically (portrait-first) which wastes horizontal space in landscape. Buttons are constrained to --btn-max-w (240-360px) which looks narrow on a 16:9 screen. The menu needs wider button constraints and better use of horizontal space.
Output: MainMenu looks proportional in landscape with properly sized buttons and spacing.
</objective>

**Tech stack available:** React 18, CSS custom properties, clamp()
**Established patterns:**
- vh-based scaling for short axis in landscape
- --overlay-max-w: clamp(280px, 45vw, 420px)
- --btn-max-w: clamp(240px, 40vw, 360px)
- --touch-target: clamp(36px, 10vmin, 52px)
- Everything inside fixed 16:9 container (#root)

**Current MainMenu layout:**
- .menu-backdrop: flex column, centered, gap var(--sp-lg)
- .menu-title: 2.5rem fixed
- .menu-play: width 100%, max-width var(--btn-max-w)
- .menu-online-row: flex row, max-width var(--btn-max-w)
- .menu-inline-section: 90% width, max-width var(--overlay-max-w)
- .menu-gear: absolute bottom-right
- .menu-help: absolute bottom-left

**What needs to change:**
- Widen --btn-max-w and --overlay-max-w for landscape proportions
- Menu title font should use responsive token (not fixed 2.5rem)
- Menu subtitle should use responsive token (not fixed 0.9rem)
- Ensure menu looks balanced in 16:9 container
</context>
