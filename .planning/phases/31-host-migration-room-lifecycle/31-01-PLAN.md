- SeatState machine: 'human-active' / 'human-afk' / 'bot'
- broadcastSeatStateChanged() for seat transitions

- Grace timer expiry calls promoteToBotFromAFK — same gap
- removePlayer() at ~line 1591 already has host migration logic (lines 1657-1667)
- startEmptyRoomKeepalive() at ~line 1687 — 10s timer, status → "closed"
- midGameJoiners map tracks joiners not yet in game

**Constraining decisions:**
- 29-01: Grace window = remaining phase timer; non-timed phases = immediate bot
- 28-02: AFK threshold = 2 consecutive timeouts → bot promotion
- 30-01: midGameJoiners tracked separately from players; pendingSeatClaims queued

**No-migrate-back rule:** When original host reconnects or takes over a bot seat, they do NOT get host back. The current host stays host. This is inherent — migrateHost sets a new hostId, and nothing restores the old one.
