**Constraining decisions:**
- PhysicsDie uses Rapier RigidBody with onSleep/onWake for settle detection
- Settings toggle pattern: Zustand store field + toggle in Settings panel
- HUD is HTML sibling to Canvas — haptic triggers from both R3F (collisions) and React (UI events)
- Performance mode exists — haptics should respect it (off in simple mode? or independent?)

**Discovery findings:**
- Vibration API: `navigator.vibrate(ms)` or `navigator.vibrate([pulse, pause, pulse, ...])`
- **NOT supported on iOS Safari** — feature detect with `'vibrate' in navigator`
- Android Chrome/Firefox: fully supported
- All haptic calls must be no-ops when unsupported (graceful degradation)
- Rapier @react-three/rapier supports `onContactForce` callback on RigidBody for collision intensity
</context>

<task type="auto">
  <name>Task 1: Create haptics utility + wire dice collision events</name>
  <files>src/utils/haptics.ts, src/components/PhysicsDie.tsx</files>
  <action>
1. **Create src/utils/haptics.ts:**
   - `isHapticsSupported()`: returns `'vibrate' in navigator`
   - `hapticPulse(ms: number)`: calls `navigator.vibrate(ms)` if supported, no-op otherwise
   - `hapticPattern(pattern: number[])`: calls `navigator.vibrate(pattern)` if supported
   - Define named pattern constants:
     - `HAPTIC_BOUNCE`: 8ms (quick tap, scales with force)
     - `HAPTIC_LOCK`: [15, 50, 15] (double tap — satisfying lock click)
     - `HAPTIC_UNLOCK`: [10, 30, 10, 30, 10] (triple light tap — release feeling)
     - `HAPTIC_ROLL_START`: 30ms (single medium pulse — "go!")
     - `HAPTIC_SCORE`: [5, 20, 5, 20, 5, 20, 10] (escalating taps — counting up)
     - `HAPTIC_WIN`: [20, 40, 20, 40, 30, 60, 50] (celebration pattern)
   - `hapticBounce(forceMagnitude: number)`: scale pulse duration by force. Clamp force to [0, 50], map to [3, 15]ms. Below force 5 = skip (too light to feel). This prevents haptic spam on tiny contacts.
   - Export all functions and patterns.

2. **Wire collision events in PhysicsDie.tsx:**
   - Import `hapticBounce` from haptics utility
   - Add `onContactForce` callback to the RigidBody component
   - In the callback: extract `totalForceMagnitude` from the event, call `hapticBounce(totalForceMagnitude)`
   - Gate behind a prop: `hapticsEnabled: boolean` (passed from parent)
   - Do NOT trigger haptics for AI dice — only player 0's pool dice. Check via a `isHumanDie` prop or similar.

Keep haptic functions pure utilities — no React state, no store dependency. The gating (enabled/disabled) happens at the call site, not inside the utility.
  </action>
