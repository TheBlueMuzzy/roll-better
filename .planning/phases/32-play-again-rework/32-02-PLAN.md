<task type="auto">
  <name>Task 2: Auto-match persistentId to old bot-held seat</name>
  <files>party/server.ts</files>
  <action>
**After sending seat_list to a mid-game joiner, check for auto-match:**

After `sendSeatList(conn)` is called for a player who entered via play_again (late joiner), check if their persistentId maps to a previous game seat:

1. Get the joiner's persistentId from `this.midGameJoiners.get(conn.id)`
2. Look up `this.previousGamePersistentIds.get(persistentId)` → returns old seatIndex (or undefined)
3. If found, check the current game: is that seatIndex occupied by a bot? (`this.gameState.players.find(p => p.seatIndex === oldSeatIndex && p.seatState === 'bot')`)
4. If the seat is a bot AND not already in `this.pendingSeatClaims`:
   - Auto-claim: add to `this.pendingSeatClaims.set(oldSeatIndex, conn.id)`
   - Send `seat_claim_result { success: true, seatIndex: oldSeatIndex }` to the joiner
   - Log: `[AUTO-MATCH] ${name} auto-claimed seat ${oldSeatIndex} (returning player)`
5. If the old seat is NOT a bot (another player took it) or already claimed:
   - Don't auto-claim — let the player manually pick from available seats (seat_list already sent)
   - Optionally log: `[AUTO-MATCH] ${name}'s old seat ${oldSeatIndex} unavailable — manual selection required`

**Also apply auto-match for normal mid-game joins (not just play_again):**
Any mid-game joiner with a persistentId matching previousGamePersistentIds should get auto-matched. This handles the case where a player disconnects, the game ends, a new game starts, and they reconnect by entering the room code fresh (not via play_again).

Move the auto-match logic to a helper method `tryAutoMatchSeat(conn)` and call it:
- After `sendSeatList(conn)` in the late play_again handler (Task 1)
- After `sendSeatList(conn)` in the existing `handleJoin` mid-game branch (around line 392)

**Edge case:** If `previousGamePersistentIds` is empty (first game, or IDs were cleared), auto-match is simply skipped. No error.

**Add auto_claimed field to SeatClaimResultMessage (protocol.ts):**
Add optional `autoMatched?: boolean` to `SeatClaimResultMessage` so the client can show appropriate feedback ("Reclaiming your seat..." vs "Seat claimed").
