<task type="auto">
  <name>Task 1: Add invisible boundary walls around rolling area</name>
  <files>src/components/RollingArea.tsx, src/components/Scene.tsx</files>
  <action>
Create `src/components/RollingArea.tsx`:

1. Calculate visible bounds from the camera setup:
   - Camera at y=12, fov=50, looking down at floor (y=0)
   - Vertical half-extent at floor: `12 * tan(25° * PI/180)` ≈ 5.6
   - But the viewport is 9:16 portrait, so horizontal is narrower: `5.6 * (9/16)` ≈ 3.15
   - Use slightly tighter bounds so dice don't scrape edges: ~2.8 horizontal, ~4.5 vertical (Z axis)
   - Store as constants: `ARENA_HALF_X = 2.8`, `ARENA_HALF_Z = 4.5`

2. Create 4 invisible wall colliders as fixed RigidBodies:
   - Left wall: CuboidCollider at x=-ARENA_HALF_X, tall (args [0.1, 3, ARENA_HALF_Z])
   - Right wall: CuboidCollider at x=+ARENA_HALF_X, same
   - Front wall: CuboidCollider at z=+ARENA_HALF_Z (args [ARENA_HALF_X, 3, 0.1])
   - Back wall: CuboidCollider at z=-ARENA_HALF_Z, same
   - All walls: `type="fixed"`, `restitution={0.3}`, NO visible mesh (invisible colliders only)
   - Wall height of 3 is enough to catch bouncing dice (they rarely go above y=2-3)

3. Export `ARENA_HALF_X` and `ARENA_HALF_Z` constants for use by DicePool.
4. Also include the floor collider and floor mesh in this component — move them out of Scene.tsx into RollingArea for cleaner organization. Keep the same floor properties: CuboidCollider [5, 0.1, 5], dark walnut mesh, receiveShadow.

Update Scene.tsx:
- Replace the inline floor RigidBody with `<RollingArea />`.
- Move the floor click handler to RollingArea (pass `onFloorClick` prop).
  </action>

<task type="auto">
  <name>Task 2: Create DicePool component with multi-dice spawning and rollAll</name>
  <files>src/components/DicePool.tsx, src/components/Scene.tsx</files>
  <action>
Create `src/components/DicePool.tsx`:

1. Props: `count: number` (how many dice), `color: string`, `onAllSettled?: (results: number[]) => void`.
2. Generate non-overlapping spawn positions for N dice:
   - Use a grid layout centered at origin: columns = ceil(sqrt(count)), spacing = 1.3 (die is 1x1, leave 0.3 gap)
   - Calculate grid positions, center the grid at (0, 0.6, 0) — slightly above floor
   - Export a `getSpawnPositions(count: number): [number, number, number][]` function for reuse
   - Add small random offset (±0.1) to each position for visual variety
3. Render `count` PhysicsDie components, each with a unique key and spawn position.
4. Store refs to each PhysicsDie using `useRef<(PhysicsDieHandle | null)[]>` — initialize as array of nulls, assign via callback refs.
5. `rollAll()` method exposed via forwardRef/useImperativeHandle:
   - Reset settled tracking
   - Call `roll()` on each PhysicsDie ref
6. Track settle state with a ref: `const settledCount = useRef(0)` and `const results = useRef<(number | null)[]>([])`.
   - Each die's `onResult` callback: store the value in results array at that die's index, increment settledCount
   - When `settledCount.current === count`: fire `onAllSettled?.(results.current as number[])`, reset settledCount to 0
7. Export `DicePoolHandle` interface: `rollAll(): void`.

Update Scene.tsx:
- Replace single PhysicsDie with `<DicePool count={5} color={PLAYER_COLORS.red} onAllSettled={handleAllSettled} />` (use 5 dice for testing — enough to see interactions without being too many).
- Move click handler: floor click calls `dicePoolRef.current.rollAll()`.
- `handleAllSettled(results)`: log results and set state for display.
- Update the result display to show all values (e.g., "Results: 3, 5, 1, 6, 2").
  </action>
