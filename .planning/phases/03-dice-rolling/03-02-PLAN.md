<task type="auto">
  <name>Task 1: Create getFaceUp utility function</name>
  <files>src/utils/diceUtils.ts</files>
  <action>
Create `src/utils/diceUtils.ts` with a `getFaceUp` function:

1. Define the 6 face normals as Vector3 constants (matching Die3D's layout):
   - Face 1: (0, 1, 0) — +Y
   - Face 6: (0, -1, 0) — -Y
   - Face 5: (1, 0, 0) — +X
   - Face 2: (-1, 0, 0) — -X
   - Face 3: (0, 0, 1) — +Z
   - Face 4: (0, 0, -1) — -Z

2. `getFaceUp(quaternion: Quaternion): number`
   - Takes a Three.js Quaternion (the die's world rotation)
   - For each face normal, create a clone, apply the quaternion rotation via `normal.clone().applyQuaternion(quaternion)`
   - Compute dot product of rotated normal with world up vector (0, 1, 0)
   - The face with the HIGHEST dot product is pointing most upward — return that face's value
   - Return type: number (1-6)

3. Use Three.js Vector3 for normals. Create them as module-level constants (not inside the function — avoid GC pressure on every call).

4. Also export the `FACE_NORMALS` array for potential reuse (array of `{value: number, normal: Vector3}`).

Do NOT allocate new Vector3 objects inside getFaceUp on every call — use a single reusable scratch Vector3 at module level for the rotation computation.
  </action>
  <verify>Import and call getFaceUp with known quaternions in a quick test: identity quaternion (no rotation) should return 1 (face 1 is +Y, pointing up by default). A 180-degree rotation around X should return 6 (bottom face now up).</verify>
  <done>getFaceUp returns correct face value for identity quaternion (1) and basic rotations. Uses pre-allocated vectors, no per-call allocations.</done>
</task>
