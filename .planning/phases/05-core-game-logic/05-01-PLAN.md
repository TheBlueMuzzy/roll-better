<task type="auto">
  <name>Task 1: Expand types and Zustand store</name>
  <files>src/types/game.ts, src/store/gameStore.ts</files>
  <action>
**Types (game.ts):**
- Keep existing GamePhase values. Add 'idle' phase (between rolls within a round ‚Äî after locking, before next roll. This is when unlock interaction happens).
  Updated flow: lobby ‚Üí rolling ‚Üí locking ‚Üí idle ‚Üí rolling... (loop until winner) ‚Üí scoring ‚Üí roundEnd ‚Üí rolling...
  'unlocking' is part of 'idle' ‚Äî the player chooses to unlock during idle, then rolls again.
- Expand `Die` interface: add `lockedToSlot: number | null` (which goal slot index 0-7 this die is locked to, null = free in pool)
- Expand `Player` interface:
  - `startingDice: number` ‚Äî how many dice this player starts the round with (modified by handicap, default 2)
  - `poolSize: number` ‚Äî current number of dice in pool (increases when unlocking)
  - `lockedDice: LockedDie[]` ‚Äî dice currently locked to goal slots
  - `selectedForUnlock: number[]` ‚Äî goal slot indices selected for unlock (toggled by player during idle phase)
  - Remove `dice: Die[]` (replace with poolSize + lockedDice approach ‚Äî physics dice are managed by DicePool component, not stored as objects)
  - Remove `handicap` field, rename to `startingDice` which IS the handicap result
- Add `LockedDie` interface: `{ goalSlotIndex: number, value: number }` ‚Äî a die locked to a specific goal slot
- Add `RoundState` interface:
  - `goalValues: number[]` ‚Äî the 8 sorted goal values for this round
  - `rollResults: number[] | null` ‚Äî latest roll results (sorted ascending)
  - `rollNumber: number` ‚Äî which roll this is in the round (starts at 1)
- Expand `GameState`:
  - Keep `phase`, `players`, `currentRound`
  - Remove `goalDice: Die[]` (replace with roundState.goalValues)
  - Remove `currentPlayerIndex` (simultaneous play ‚Äî all players roll at once)
  - Add `roundState: RoundState`
  - Add `sessionTargetScore: number` (default 20)

**Store (gameStore.ts):**
- Keep `reset()` and `setPhase()`
- Add `initGame(playerCount: number)` ‚Äî creates players with default names ("You", "Player 2"...), colors from PLAYER_COLORS, score 0, startingDice 2, isAI flags (player 0 = human, rest = AI)
- Add `initRound()` ‚Äî generates new goal (8 random 1-6 sorted ascending), resets pools to startingDice, clears locks, sets phase to 'rolling', increments currentRound
- Add `setRollResults(results: number[])` ‚Äî stores roll results, sets phase to 'locking'
- Add `lockDice(playerIndex: number, locks: LockedDie[])` ‚Äî adds locked dice to player, reduces poolSize by locks.length, sets phase to 'idle'
- Add `toggleUnlockSelection(playerIndex: number, goalSlotIndex: number)` ‚Äî toggles a slot index in selectedForUnlock array
- Add `confirmUnlock(playerIndex: number)` ‚Äî for each selected slot: remove from lockedDice, add 1 to poolSize (returned die) + add 1 to poolSize (bonus die from goal). Clear selectedForUnlock. Set phase to 'rolling'.
- Add `skipUnlock(playerIndex: number)` ‚Äî clear selectedForUnlock, set phase to 'rolling'
- Add `scoreRound()` ‚Äî calculates score for each player who completed the goal (8 - max(0, (poolSize - 8) * 2), floor 0), adds to player.score, sets phase to 'scoring'
- Add `applyHandicap()` ‚Äî for each player: won this round ‚Üí startingDice = max(1, startingDice - 1), lost ‚Üí startingDice = min(12, startingDice + 1). Set phase to 'roundEnd'.
- Add `checkWinner()` ‚Äî returns true if any player has all 8 goal slots locked (lockedDice.length === 8)
- Add `checkSessionEnd()` ‚Äî returns true if any player.score >= sessionTargetScore

Do NOT import PLAYER_COLORS from Die3D into the store ‚Äî define a simple color array directly in the store file to avoid circular dependency. Use the same hex values: red '#e74c3c', blue '#3498db', green '#2ecc71', yellow '#f1c40f', purple '#9b59b6', orange '#e67e22', pink '#e91e8f', cyan '#1abc9c'.

  <files>src/App.tsx, src/components/Scene.tsx, src/components/HUD.tsx</files>

**App.tsx:**
- Import `useGameStore` and read state from store instead of local useState
- Remove `useState` for `isRolling` and `diceResults` ‚Äî read from store: `isRolling` = `phase === 'rolling'`, `diceResults` = `roundState.rollResults`
- On mount (useEffect), call `initGame(2)` to create a 2-player game (1 human + 1 AI), then `initRound()` to generate first goal
- `handleRollStart`: call `setPhase('rolling')`
- `handleResults`: call `setRollResults(results)` ‚Äî this stores results and transitions to 'locking'
- `handleRoll`: only allow rolling when phase is 'idle' or 'rolling' ‚Äî guard with `if (phase !== 'idle' && phase !== 'lobby') return`
  Actually, simpler: the initial state after initRound is 'rolling' which is wrong ‚Äî the player hasn't rolled yet. Use a new approach:
  - After `initRound()`, phase should be 'idle' (waiting for player to tap roll)
  - When player taps "Tap To Roll": set phase to 'rolling', trigger rollAll
  - When results come back: setRollResults stores them, phase ‚Üí 'locking'
  - After auto-lock logic runs (Phase 5 plan 3): phase ‚Üí 'idle'
  - For now (this plan), after 'locking' just immediately go to 'idle' so player can roll again
- Pass store values to HUD: score from `players[0].score`, round from `currentRound`

**Scene.tsx:**
- Import `useGameStore` and read `roundState.goalValues`, `players[0]` for color/locked/stats
- Replace TEST_GOAL_VALUES with `roundState.goalValues` (fall back to empty array if no round state)
- Replace TEST_LOCKED_VALUES with a computed array from `players[0].lockedDice`: create 8-slot array, fill locked slots with die values
- Replace TEST_PLAYER with store player data
- Replace TEST_DICE_COUNT with `players[0].poolSize`
- Remove all TEST_ constants

**HUD.tsx:**
- Update to read from store directly using `useGameStore()` instead of props
- Remove all props except `onRoll` (still needs callback to trigger Scene ref)
- Read phase, score, round, rollResults from store
- Status text logic:
  - phase === 'rolling' ‚Üí "Rolling"
  - phase === 'locking' ‚Üí show roll results (comma separated)
  - phase === 'idle' ‚Üí "Tap To Roll"
  - phase === 'lobby' ‚Üí "Starting..."
- Guard onRoll: only fire when phase === 'idle'
- Show pool stats: X (poolSize) / Y (12) / Z (startingDice)

Keep the existing handleFloorClick in Scene ‚Äî it still triggers rolls via DicePool ref.
