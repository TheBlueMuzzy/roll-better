**Constraining decisions:**
- Goal is 8 sorted values (e.g., [1,1,2,2,3,4,5,6])
- Roll results are sorted ascending
- Lock limit per value = count of that value in goal
- Already-locked dice count against the limit
- Example: goal has 2 twos, 1 already locked → only 1 more two can lock
- Extra matching dice beyond the limit stay in pool (NOT locked)
- Results sorted ascending from Phase 3
</context>

<feature>
  <name>Match Detection + Auto-Lock Algorithm</name>
  <files>src/utils/matchDetection.ts, src/utils/matchDetection.test.ts</files>
  <behavior>
    Function signature: `findAutoLocks(goalValues: number[], rolledValues: number[], currentLocks: LockedDie[]) → LockedDie[]`

    Where `LockedDie = { goalSlotIndex: number, value: number }`

    **Rules:**
    1. For each rolled die value, check if any goal slot with that value is still available (not already locked by currentLocks)
    2. A goal slot is "available" if: goalValues[slotIndex] === dieValue AND no entry in currentLocks has goalSlotIndex === slotIndex
    3. When multiple rolled dice match the same value, fill available slots left-to-right (lowest index first)
    4. Return only NEW locks (don't include currentLocks in output)
    5. If no matches found, return empty array

    **Test cases (input → expected output):**

    Case 1 — Basic matching, no existing locks:
      goal: [1,1,2,2,3,4,5,6], rolled: [1,3,5], locks: []
      → [{goalSlotIndex: 0, value: 1}, {goalSlotIndex: 4, value: 3}, {goalSlotIndex: 6, value: 5}]

    Case 2 — Multiple dice matching same value:
      goal: [1,1,2,2,3,4,5,6], rolled: [2,2,2], locks: []
      → [{goalSlotIndex: 2, value: 2}, {goalSlotIndex: 3, value: 2}]
      (only 2 twos in goal, third 2 stays in pool)

    Case 3 — Existing locks reduce available slots:
      goal: [1,1,2,2,3,4,5,6], rolled: [2,2,2], locks: [{goalSlotIndex: 2, value: 2}]
      → [{goalSlotIndex: 3, value: 2}]
      (slot 2 already locked, only slot 3 available)

    Case 4 — No matches:
      goal: [1,1,2,2,3,4,5,6], rolled: [6,6,6], locks: [{goalSlotIndex: 7, value: 6}]
      → [] (only one 6 in goal, already locked)

    Case 5 — Perfect match (all 8):
      goal: [1,1,2,2,3,4,5,6], rolled: [1,2,3,4,5,6], locks: [{goalSlotIndex: 0, value: 1}, {goalSlotIndex: 2, value: 2}]
      → [{goalSlotIndex: 1, value: 1}, {goalSlotIndex: 3, value: 2}, {goalSlotIndex: 4, value: 3}, {goalSlotIndex: 5, value: 4}, {goalSlotIndex: 6, value: 5}, {goalSlotIndex: 7, value: 6}]

    Case 6 — Empty roll:
      goal: [1,1,2,2,3,4,5,6], rolled: [], locks: []
      → []

    Case 7 — All slots already locked:
      goal: [1,1,2,2,3,4,5,6], rolled: [1,2,3], locks: [all 8 slots filled]
      → []
  </behavior>
  <implementation>
    Create `src/utils/matchDetection.ts` with:

    1. Build a "remaining slots" map from goal: for each unique value, list the slot indices not yet locked
    2. Iterate through rolled values (sorted), for each:
       - Look up remaining slots for that value
       - If any available, take the first (lowest index), add to result, remove from remaining
       - If none available, skip (die stays in pool)
    3. Return the result array sorted by goalSlotIndex

    Algorithm is O(n*m) where n=rolled dice, m=goal slots — both max 12, so trivial performance.

    Import the LockedDie type from types/game.ts.
  </implementation>
</feature>
