**HUD.tsx — phase-aware display:**
- Read game state from store: `const { phase, currentRound, players, roundState, sessionTargetScore } = useGameStore()`
- Get human player: `const player = players[0]`
- Status text by phase:
  - 'lobby': "Starting..."
  - 'idle': "Tap To Roll" (tappable)
  - 'rolling': "Rolling..." (not tappable)
  - 'locking': show newly locked count, e.g., "Locked 2!" or "No matches" — brief display before going to idle
  - 'scoring': "Round Complete! +{score}pts"
  - 'roundEnd': "Next Round..."
  - 'sessionEnd': "Game Over! Score: {score}"
- Top bar: "Round {currentRound}" on left, "{score} / {targetScore}" on right
- Bottom area (below status text): Pool stats line: "{poolSize}/12 dice · Start: {startingDice}"
- Only allow tap-to-roll when phase === 'idle'

**App.tsx — roll guarding:**
- `handleRoll` should check `useGameStore.getState().phase === 'idle'` before triggering
- If phase is not 'idle', ignore the tap (no-op)
- Ensure handleFloorClick in Scene also respects this guard

**Visual polish (minimal, no animations yet — that's Phase 6):**
- When in 'locking' phase, show lock count for 1 second, then transition to 'idle'
  Use setTimeout in store or in a useEffect in App watching phase === 'locking'
- When 'scoring', show score for 2 seconds before transitioning to next round
- When 'sessionEnd', show final score and stop the game loop
