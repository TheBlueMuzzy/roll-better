**Constraining decisions:**
- Phase 5: findAutoLocks is the lock matching algorithm (left-to-right slot filling)
- Phase 5: Scoring = max(0, 8 - poolSize * 2) — larger pool = worse score
- Phase 5: Must-unlock rule: poolSize=0 AND lockedDice.length<8 → must unlock at least 1
- Phase 5: 12-die cap: poolSize + numUnlocks*2 ≤ 12
- Phase 5: Each unlock returns the locked die + 1 bonus die (2 dice added to pool per unlock)

**Game context for AI strategy:**
- Goal has 8 sorted values (1-6). Player rolls poolSize dice, matching values auto-lock.
- After locking, player can unlock 0+ locked dice. Each unlock returns 2 dice to pool.
- Unlocking increases pool (more dice to match with) but tanks score (-2 per extra die).
- The strategic tension: unlock to match faster vs keep pool small for higher score.
- A player with 2 dice rolling against 6 remaining goal slots has low match probability.
- A player with 8 dice rolling against 6 remaining slots has high match probability but scores 0 if they win.

    **Easy strategy:**
    - 40% chance to unlock anything at all (60% skip)
    - If unlocking: picks 1 random locked die
    - Exception: always respects must-unlock rule (poolSize=0, locked<8)
    - Exception: always respects 12-die cap

    **Medium strategy:**
    - Considers remaining unmatched goal slots count
    - Considers current pool size vs remaining slots
    - Heuristic: unlock if poolSize < remainingSlots / 2 (too few dice to match efficiently)
    - Picks dice whose values appear MOST frequently in remaining goal (maximize match probability after unlock)
    - Unlocks 1-2 dice max
    - Respects must-unlock and 12-die cap

    **Hard strategy:**
    - Calculates expected matches: for each possible roll of poolSize dice, estimate how many would match remaining goal
    - Only unlocks when expected match rate is poor (< 0.5 matches per roll with current pool)
    - When unlocking: picks dice whose values appear LEAST in remaining goal (sacrifice low-probability matches to gain dice)
    - Unlocks minimum needed to improve expected match rate above threshold
    - Never unlocks if already near completion (≤ 2 remaining slots)
    - Respects must-unlock and 12-die cap

    **Test cases (input → expected output patterns):**
    - must-unlock (poolSize=0, 4 locked) → returns at least 1 slot (all difficulties)
    - 12-die cap (poolSize=10, 2 locked) → returns empty (can't unlock without exceeding 12)
    - no locked dice → returns empty (nothing to unlock)
    - all 8 locked → returns empty (already won, shouldn't reach this state)
    - easy: poolSize=4, 3 locked → returns 0 or 1 slots (probabilistic, but never >1)
    - medium: poolSize=1, 5 locked → returns 1-2 slots (pool too small for 3 remaining)
    - medium: poolSize=6, 2 locked → returns empty (pool large enough for 6 remaining)
    - hard: 1 remaining slot, poolSize=2 → returns empty (close to winning, don't unlock)
    - hard: 6 remaining slots, poolSize=2 → returns 1+ slots (need more dice)
  </behavior>
