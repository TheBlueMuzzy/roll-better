# Existing deployment:
GitHub Pages already auto-deploys on push to master via .github/workflows/deploy.yml.
PartyKit server deploys separately via `npx partykit deploy`.
Base path is `/roll-better/` in production (set in vite.config.ts).
VITE_PARTY_HOST set to `roll-better.thebluemuzzy.partykit.dev` in the GitHub Actions workflow.

# PWA reference (from Context7 — vite-plugin-pwa):
- Install: `npm install -D vite-plugin-pwa`
- Import: `import { VitePWA } from 'vite-plugin-pwa'`
- Key config: `registerType: 'autoUpdate'` for automatic SW updates
- Workbox `globPatterns` for precaching: `['**/*.{js,css,html,ico,png,svg,woff2}']`
- Manifest goes in plugin config (not a separate file)
- Icons: 192x192 and 512x512 PNGs in public/ folder
- React integration: `import { registerSW } from 'virtual:pwa-register'` (auto-register, no React hook needed for autoUpdate)

**Constraining decisions:**
- No accounts, no user data — PWA is purely for installability and app shell caching
- WebSocket connections to PartyKit must NOT be intercepted by service worker
- Game works fine without offline support (multiplayer requires network) — SW just caches the app shell for fast loading

<task type="auto">
  <name>Task 1: Install vite-plugin-pwa and configure manifest + service worker</name>
  <files>package.json, vite.config.ts</files>
  <action>
Install vite-plugin-pwa as a dev dependency:
```
npm install -D vite-plugin-pwa
```

Update vite.config.ts to add VitePWA plugin with this configuration:
- `registerType: 'autoUpdate'` — new service workers activate immediately, no prompt needed
- `manifest` object with:
  - `name: 'Roll Better'`
  - `short_name: 'Roll Better'`
  - `description: 'Multiplayer dice-matching game'`
  - `theme_color: '#1a1a2e'` (dark background matching the game's dark wood theme)
  - `background_color: '#1a1a2e'`
  - `display: 'standalone'`
  - `orientation: 'portrait'`
  - `start_url: './'` (relative — works with any base path)
  - `scope: './'`
  - `icons` array with 192x192 and 512x512 entries (files created in Task 2)
- `workbox` object with:
  - `globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2,wasm}']` — include .wasm for Rapier physics
  - `navigateFallback: 'index.html'` — SPA fallback
  - `navigateFallbackDenylist: [/^\/api/]` — safety exclusion
  - `runtimeCaching` array with ONE entry: pattern `/(partykit|parties)\//` handler `'NetworkOnly'` — ensures WebSocket/PartyKit traffic is never cached

Do NOT use `includeAssets` for the icons — they'll be in the manifest and precached automatically.

Keep the existing `base` path logic and `react()` plugin. Add VitePWA after react() in the plugins array.
  </action>
  <verify>
`npm run build` succeeds and produces `dist/sw.js` (service worker) and `dist/manifest.webmanifest` in the output.
Check: `ls dist/sw.js dist/manifest.webmanifest` both exist after build.
  </verify>
  <done>vite-plugin-pwa installed, vite.config.ts updated, build produces service worker + manifest</done>

<task type="auto">
  <name>Task 2: Create icon assets and update index.html meta tags</name>
  <files>public/favicon.svg, public/pwa-192x192.png, public/pwa-512x512.png, index.html</files>
  <action>
**Create favicon.svg** in public/:
A simple dice-themed SVG icon — a rounded square (#1a1a2e dark background) with a white dot pattern resembling a die face (showing 5 — the classic dice icon). Keep it minimal: viewBox 0 0 512 512, rounded rect with rx="64", five circles for pips.

**Generate PNG icons** from the SVG:
Use a quick Node script (run once, then delete) to convert the SVG to 192x192 and 512x512 PNGs using the `sharp` library:
```
npm install -D sharp
```
Write a script `scripts/generate-icons.js` that:
1. Reads `public/favicon.svg`
2. Renders to `public/pwa-192x192.png` at 192x192
3. Renders to `public/pwa-512x512.png` at 512x512
4. Run it: `node scripts/generate-icons.js`
5. After generating, remove the script and uninstall sharp: `npm uninstall -D sharp && rm -rf scripts/`

If sharp has install issues on Windows, use an alternative approach: create the PNGs directly with sharp's SVG-to-PNG, or create them as simple canvas-drawn images. The icons are placeholders — Muzzy will replace them with proper art later.

**Update index.html** `<head>` section — add these tags (keep existing apple-mobile-web-app tags):
- `<meta name="theme-color" content="#1a1a2e" />` — matches manifest theme_color
- `<link rel="icon" href="/favicon.svg" type="image/svg+xml" />` — modern browsers
- `<link rel="apple-touch-icon" href="/pwa-192x192.png" />` — iOS home screen icon
- `<meta name="description" content="Multiplayer dice-matching game" />` — for SEO/share previews

Do NOT add a `<link rel="manifest">` tag — vite-plugin-pwa injects it automatically during build.

Note: favicon.svg href should NOT include the base path — Vite handles base path rewriting for assets in public/.
  </action>
  <verify>

3. `grep "theme-color" index.html` — meta tag present
4. `grep "apple-touch-icon" index.html` — link tag present
  </verify>
