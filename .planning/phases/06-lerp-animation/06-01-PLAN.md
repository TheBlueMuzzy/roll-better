**Layout reference (confirmed from source):**
- GoalRow default z = -4.67, PlayerRow default z = -3.77
- Slot X: `getSlotX(i) = (i - 3.5) * 0.7` (exported from GoalRow.tsx)
- DIE_SIZE ≈ 0.659 (exported from RollingArea.tsx)
- Rolling area: X ±2.8, Z from -1.7 to 4.5
- Die Y at rest: DIE_SIZE / 2 ≈ 0.33

**Established patterns:**
- DicePool generation-counter keys force correct remount after pool shrinks
- Phase transitions: chained useEffect timers (1s locking, 2s scoring, 1.5s roundEnd) in App.tsx
- useFrame + ref mutation for per-frame animation (R3F golden rule — NEVER React state for per-frame)
- getRotationForFace(value) maps face value → euler rotation (GoalRow.tsx)
- GoalRow and PlayerRow render OUTSIDE Physics; DicePool inside Physics

**Known concern:**
- BUG-001/ISS-002: Face misread from canted dice. If a die's face is misread, the lock lerp animates the "wrong" die. Pre-existing physics bug — not caused by this plan.

  <action>
  **AnimatingDie.tsx (new file):**
  Component that renders a Die3D at an interpolated position using useFrame.
  - Props: `fromPos: [x,y,z]`, `toPos: [x,y,z]`, `value: number`, `color: string`, `duration?: number` (default 0.6), `onComplete?: () => void`
  - Use a `groupRef` (THREE.Group) + `useFrame` for position interpolation — mutate ref, NOT React state
  - Track elapsed time in a ref. Each frame: `t = clamp(elapsed / duration, 0, 1)`
  - Ease-out cubic: `eased = 1 - Math.pow(1 - t, 3)`
  - Interpolate X and Z linearly: `from + (to - from) * eased`
  - Add Y arc for physical feel: `baseY + Math.sin(t * Math.PI) * 0.8` (0.8 unit parabolic arc above the straight line — die lobs upward during flight)
  - Set rotation using `getRotationForFace(value)` — die shows correct face throughout flight
  - When `t >= 1`, call `onComplete()` once (hasFired ref guard to prevent double-call)
  - Render: `<group ref={groupRef} scale={DIE_SIZE}><Die3D color={color} /></group>`
  - Import DIE_SIZE from RollingArea, getRotationForFace from GoalRow, Die3D
