**Current phase transition timing (App.tsx useEffect chain):**
- Locking phase: 1000ms delay → check winner
- Scoring phase: 2000ms delay → applyHandicap → set 'roundEnd'
- RoundEnd phase: 1500ms delay → initRound → set 'idle'

**Score calculation:**
- roundScore = max(0, 8 - poolSize * 2) — stored temporarily during scoring phase
- Player score updates via scoreRound() which adds roundScore to player.score

**Handicap:**
- Winner: startingDice -= 1 (min 1)
- Loser: startingDice += 1 (max 12)
- Applied during applyHandicap() between scoring and roundEnd

**Goal row:**
- GoalRow renders 8 static Die3D at z = -4.67
- Values set by initRound → roundState.goalValues
- Currently instant — values just swap when initRound fires

**HUD:**
- HTML overlay (sibling to Canvas, not inside R3F)
- Shows phase-dependent status text
- During scoring: "Round Complete! +{roundScore}pts"
- Score display at top: "{score}/{target}"

**PlayerIcon:**
- R3F component at lower-left of rolling area
- Shows name, score badge, pool/matches/Z stats
- Position: [-ARENA_HALF_X + 0.3, 0, ROLLING_Z_MAX - 0.3]

  **HUD.tsx — Score counting animation:**
  When phase enters 'scoring', animate the displayed score from currentScore to currentScore + roundScore over ~1.5s (leaving 0.5s of the 2s scoring window for the final number to settle visually).

  Implementation approach (HTML/DOM, not R3F):
  - Use a ref to store the display value and an animating flag
  - On phase change to 'scoring': read the previous score (before scoreRound applied it) and the roundScore
  - Use requestAnimationFrame loop: increment display value with ease-out timing
  - Ease-out: increment fast at start, slow at end. Simple approach: linear step with decreasing increment, or use `Math.ceil(remaining * 0.15)` per frame for natural deceleration
  - Display: show the counting number in the score area, maybe with a slight CSS scale pulse at final value
  - When count reaches target, stop animation, show final score

  **To get "previous score" for counting:** The store's scoreRound() currently adds roundScore to player.score immediately. Options:
  - Add `previousScore` field to store that's set before scoreRound runs — cleanest
  - OR read score before the scoring phase starts and cache it in HUD via useRef
  - Choose the simpler approach. Caching in a ref when phase transitions to 'scoring' is simplest — no store change needed. But the score may already be updated by the time HUD re-renders. So: add `roundScore` to roundState (if not already there) and compute `displayScore = player.score - roundScore` as the starting point.

  Check if roundScore is already accessible in the store. From 05-03 summary: "lastLockCount" is in roundState. Check if roundScore is stored. If not, add it to roundState in gameStore.ts and set it in scoreRound().

  **PlayerIcon.tsx — Handicap (Z) scale pop:**
  When startingDice changes (after applyHandicap), play a brief scale-pop animation on the Z display element.

  Implementation:
  - Track previous startingDice in a ref
  - In useFrame: if startingDice !== prevRef, trigger pop animation
  - Pop: scale the Z text/badge from 1.0 → 1.4 → 1.0 over 0.4s (ease-out bounce)
  - Use a `popTimer` ref: when triggered, set to 0.4, decrement each frame, compute scale from remaining time
  - Scale curve: `1.0 + 0.4 * Math.sin(Math.PI * (popTimer / 0.4))` — peaks at midpoint
  - Apply scale to the Z display group/mesh only (not the whole PlayerIcon)

  **Important:** PlayerIcon is R3F, so use useFrame + ref mutation. HUD is HTML, so use requestAnimationFrame or useEffect.

  <name>Task 2: Goal row transition animation</name>
  <files>src/components/GoalRow.tsx, src/store/gameStore.ts, src/types/game.ts, src/App.tsx</files>
  <action>
  **Goal transition concept:** During roundEnd phase, old goal dice slide right off screen (staggered per die). During the transition to idle, new goal dice slide in from the left (staggered). This uses the existing 1500ms roundEnd delay.

  **game.ts:**
  Add to RoundState: `goalTransition: 'none' | 'exiting' | 'entering'` (default 'none')

  **gameStore.ts:**
  - Initialize `goalTransition: 'none'` in roundState
  - Add `setGoalTransition(state: 'none' | 'exiting' | 'entering')` action
  - Reset to 'none' in initRound (after new goal values are set)

  **App.tsx — modify roundEnd timing:**
  Currently: single 1500ms timeout → initRound → setPhase('idle')

  Change to staged sequence:
  1. Phase becomes 'roundEnd' → immediately set goalTransition to 'exiting'
  2. After 500ms: call initRound (sets new goalValues) + set goalTransition to 'entering'
  3. After 1000ms more (1500ms total): set goalTransition to 'none' + setPhase('idle')

  This gives 500ms for exit, instant swap, 500ms for enter, then settle.

  Use chained timeouts with cleanup (same pattern as existing phase transitions — return cleanup function from useEffect). Store timeout IDs for proper cleanup on unmount/phase change.

  **GoalRow.tsx — animate dice based on goalTransition:**
  Accept new prop `transition?: 'none' | 'exiting' | 'entering'` (default 'none').

  When transition is 'none': render normally (current behavior).

  When transition is 'exiting':
  - Each die animates: X slides right by +8 units (off screen), Z-axis rotates (rolling motion)
  - Staggered: die[0] starts immediately, die[1] after 30ms, die[2] after 60ms, etc.
  - Duration: ~400ms per die (all finish within 500ms window)
  - Use useFrame + refs for animation. Track per-die progress in a ref array.
  - Compute X offset: `8 * easeIn(t)` where easeIn = `t * t` (accelerating exit)
  - Compute Z rotation: `t * Math.PI * 2` (one full roll during exit)

  When transition is 'entering':
  - Each die starts at X offset -8 (off screen left) and slides to final position
  - Staggered: same 30ms spacing
  - Duration: ~400ms per die
  - Compute X offset: `-8 * (1 - easeOut(t))` where easeOut = `1 - (1-t)^2`
  - Compute Z rotation: `(1-t) * Math.PI * 2` (rolling in, decelerating)

  **Implementation approach for GoalRow animation:**
  - Wrap each die in a `<group>` with a ref
  - In useFrame: if transition !== 'none', compute per-die offsets and apply to refs
  - When transition is 'none', ensure all offsets are 0 (reset positions)
  - Use a single useFrame that loops over all 8 die refs

  **Perf note:** 8 refs in one useFrame is fine. Don't create 8 separate useFrame hooks.

  **Pass transition prop from Scene.tsx:**
  Read `goalTransition` from store and pass to `<GoalRow values={...} transition={goalTransition} />`.
