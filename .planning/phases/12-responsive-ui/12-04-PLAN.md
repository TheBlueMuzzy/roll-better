  <action>
These components use inline styles (not CSS classes) because they're drei Html elements positioned in 3D space. Replace hardcoded px values with viewport-relative calculations using a shared scale factor.

**In PlayerProfileGroup.tsx:**

Add a responsive scale helper at the top of the component (inside the function body):
```
const vw = typeof window !== 'undefined' ? window.innerWidth : 390;
const scale = Math.min(Math.max(vw / 390, 0.85), 1.3);
```
This gives scale=1.0 at 390px (iPhone 14 baseline), 0.96 at 375px (iPhone SE), 1.3 cap at 507px+.

Then multiply key dimension values by `scale`:
- Avatar width/height: 57 → Math.round(57 * scale)
- Score display width/height/fontSize: 52 → Math.round(52 * scale)
- Star emoji fontSize: 24 → Math.round(24 * scale)
- Score number fontSize: 20 → Math.round(20 * scale)
- Stats fontSize: 12 → Math.round(12 * scale)
- Gap values: 6 → Math.round(6 * scale)
- Divider fontSize: 12 → Math.round(12 * scale)

**In PlayerIcon.tsx:**

Same scale helper:
```
const vw = typeof window !== 'undefined' ? window.innerWidth : 390;
const scale = Math.min(Math.max(vw / 390, 0.85), 1.3);
```

Then multiply:
- Container padding: scale the px values
- Dot width/height: 12 → Math.round(12 * scale)
- Name fontSize: 13 → Math.round(13 * scale)
- Score fontSize: 12 → Math.round(12 * scale)
- Stats fontSize: 10 → Math.round(10 * scale)
- Gap: 6 → Math.round(6 * scale)

**Important:** Do NOT add window resize listeners or React state for this — these components remount on game state changes anyway, and adding resize state would cause unnecessary re-renders in the 3D scene. The scale is read once on mount which is sufficient since the game is portrait-locked.

Do NOT change the drei Html zIndexRange, 3D positioning, or component structure.
