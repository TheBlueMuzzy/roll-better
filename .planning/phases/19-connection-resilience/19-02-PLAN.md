<objective>
Implement client-side reconnection handling so the game recovers gracefully when a player's connection drops and PartySocket auto-reconnects.

Purpose: Without this, a temporary network drop during a game wipes all client state (useRoom.onclose calls resetState), shows "Connection lost", and kicks the player to a broken lobby. With this plan, the game stays on screen, shows "Reconnecting...", and syncs back up when the connection restores.
Output: Full client-side reconnection flow — game persists through network drops, state syncs from server on rejoin.

**Constraining decisions from 19-01:**
- Server sends `connected` ack followed by `rejoin_state` on reconnection
- Server sends `player_reconnected` to other clients
- Stable client ID via sessionStorage ensures same conn.id across reconnects
- PartySocket auto-reconnects underlying WebSocket — fires `onclose` then `onopen` on the PartySocket instance

**Key architecture facts (from full code review):**
- `useRoom` sets `socket.onclose` which calls `resetState()` — wipes roomCode, playerId, players, isConnected, gameStartData
- `useRoom` does NOT set `socket.onopen` — no handler for reconnection
- During game, `isOnlineGame` is true in Zustand store, `getGameSocket()` returns the PartySocket
- `useOnlineGame` uses `socket.addEventListener("message", handler)` — separate from useRoom's `socket.onmessage`
- On reconnect, server sends `connected` ack. useRoom's handler checks `pendingJoinRef.current` — it's null during reconnect, so no `join` is sent. This is correct because server already handled rejoin in onConnect.
- `clearOnlineMode()` wipes isOnlineGame, onlinePlayerId, etc. — only called on intentional menu exit

**NOTE: PartySocket buffers sends during disconnect.**
While disconnected, the watchdog in useOnlineGame may queue `phase_sync_request` and `skip_unlock` messages via `sendMessage()`. PartySocket buffers these and fires them all on reconnect. The server should handle these gracefully — the rejoin protocol already supersedes phase_sync, and duplicate skip_unlock is harmless. No client-side fix needed, but be aware of this behavior.

**useOnlineGame.ts — Handle rejoin_state:**
Add a new case to the message switch inside the useEffect handler:

```js
case "rejoin_state": {
  console.log("[useOnlineGame] rejoin_state — restoring game state", {
    phase: msg.phase,
    round: msg.round,
    goalValues: msg.goalValues,
    players: msg.players?.length,
  });

  const store = useGameStore.getState();

  // Apply goal values (in case round changed while disconnected)
  if (msg.goalValues) {
    store.setGoalValues(msg.goalValues);
  }

  // Sync all player states from server snapshot
  if (msg.players) {
    store.syncAllPlayerState(msg.players);
  }

  // Clear ALL animation state (we missed the animations while disconnected)
  store.clearLockAnimations();
  store.clearAILockAnimations();
  store.clearUnlockAnimations();
  store.clearAIUnlockAnimations();

  // Clear any deferred phase polling
  if (deferredPhaseInterval) { clearInterval(deferredPhaseInterval); deferredPhaseInterval = null; }
  if (deferredPhaseTimeout) { clearTimeout(deferredPhaseTimeout); deferredPhaseTimeout = null; }

  // Reset buffering flags
  store.setLocalPlayerLocked(false);
  store.setHasSubmittedUnlock(false);

  // Force phase to match server
  const rejoinPhase = msg.phase as GamePhase;
  if (store.phase !== rejoinPhase) {
    console.log(`[useOnlineGame] rejoin phase sync: "${store.phase}" -> "${rejoinPhase}"`);
    store.setPhase(rejoinPhase);
  }

  // Sync round number — player may have missed entire rounds while disconnected.
  // currentRound is displayed in HUD ("Round X") and WinnersScreen ("Rounds played: X").
  // Without this, the HUD shows a stale round number after reconnection.
  store.setCurrentRound(msg.round);

  // Reset watchdog state so it doesn't self-heal based on stale pre-disconnect timing.
  // These are `let` variables in the same useEffect closure, so they're accessible here.
  // Without this, the watchdog could detect a "stall" from before the disconnect
  // and force-reset the client to idle BEFORE rejoin_state is fully applied.
  lastPhase = rejoinPhase;
  phaseEnteredAt = Date.now();
  watchdogFireCount = 0;

  break;
}
```
