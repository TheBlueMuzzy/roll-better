**Server onConnect — rejoin detection:**
Modify `onConnect(conn)` to detect a returning player BEFORE the normal flow:

```
1. If this.gameState exists AND this.status is "playing" (or "waiting_for_rejoin" — see keepalive below):
   a. Find gamePlayer in this.gameState.players where p.id === conn.id AND p.isOnline === false AND p.intentionalLeave !== true
   b. If found:
      - Set gamePlayer.isOnline = true
      - Re-add to this.players Map with a RoomPlayer entry (name/color from gamePlayer, isHost = conn.id === this.hostId, isReady = true)
      - If this.status was "waiting_for_rejoin", change back to "playing"
      - Cancel keepalive timer if running
      - Send "connected" ack to conn (normal flow — useRoom needs this)
      - Send "rejoin_state" message to conn with: phase, round (currentRound), goalValues, players (buildPlayerSnapshot())
      - Broadcast "player_reconnected" { playerId: conn.id, playerName: gamePlayer.name } to all EXCEPT conn
      - If game is in "rolling" phase and this player hasn't rolled yet (not in rollRequestedBy), they need to roll — the phase field in rejoin_state tells them
      - If game is in "unlocking" phase and this player hasn't responded yet (not in unlockResponses), they need to unlock — same logic
      - Log "[REJOIN] Player {name} reconnected"
      - RETURN (skip normal onConnect flow)
```

**ServerPlayerState — add intentionalLeave flag:**
Add `intentionalLeave?: boolean` to the `ServerPlayerState` interface. Default undefined/false.

**removePlayer — track intentional vs disconnect:**
Add second parameter: `removePlayer(connectionId: string, intentional: boolean = false)`
- If `intentional && this.gameState`: set `gamePlayer.intentionalLeave = true` on the matching game player
- Update the "leave" message handler to call `this.removePlayer(sender.id, true)`
- `onClose` already calls `this.removePlayer(conn.id)` — defaults to `false` (disconnect, rejoin allowed)

**Room keepalive grace period:**
Add a `private keepaliveTimer: ReturnType<typeof setTimeout> | null = null` field.

In `removePlayer`, after deleting from `this.players`:
- If `this.players.size === 0 && this.gameState && this.status === "playing"`:
  - Instead of setting `this.status = "closed"`, set `this.status = "waiting_for_rejoin"` (add to RoomStatus if needed — but this is server-internal, doesn't need to be in protocol)
  - Start a 60-second keepalive timer: `this.keepaliveTimer = setTimeout(() => { this.status = "closed"; this.gameState = null; this.cleanupTimers(); }, 60_000)`
  - Log "[KEEPALIVE] Room empty — waiting 60s for rejoin"
- In onConnect rejoin: if `this.status === "waiting_for_rejoin"`, cancel keepalive timer, restore to "playing"

**Existing status check in onConnect:**
- Currently rejects if `this.status === "closed"`. Add: also allow `"waiting_for_rejoin"` for rejoin candidates (the rejoin check above runs first, so non-matching connections would fall through to the full check)
- For non-rejoin connections during "waiting_for_rejoin": reject with "Game in progress"

**cleanupTimers:**
Add keepalive timer cleanup to `cleanupTimers()`.

**Do NOT change the server-internal status type.** Use a simple string comparison. The RoomStatus in protocol.ts is for client-facing messages only.
