**Design decisions for client utility:**
- Thin wrapper around PartySocket — don't over-abstract
- Connection state tracking (connecting, connected, disconnected, error)
- Typed message send/receive using protocol types
- NOT a React hook yet — just a utility class/module. React integration comes in Phase 15 (lobby UI).
- In dev: host is localhost:1999. In production: the deployed Partykit URL.
- Room ID passed at connection time (will be the 4-letter code from Phase 15).

**PartySocket client patterns (from Context7):**
- `new PartySocket({ host, room })` — creates WebSocket connection
- `ws.addEventListener("message", handler)` — receive messages
- `ws.send(JSON.stringify(message))` — send messages
- PartySocket auto-reconnects by default
- `ws.close()` — explicit disconnect
</context>

  <action>
  Create `src/utils/partyClient.ts` — a simple, non-React utility for connecting to the Partykit server.

  1. **Configuration:**
     - `PARTY_HOST` constant: `localhost:1999` in dev, configurable for production
     - Use `import.meta.env.VITE_PARTY_HOST` with fallback to `localhost:1999`
     - Add `VITE_PARTY_HOST` to a `.env.example` for documentation

  2. **createPartyConnection(roomId: string) function:**
     - Creates and returns a `PartySocket` instance: `new PartySocket({ host: PARTY_HOST, room: roomId })`
     - Returns the socket — caller manages lifecycle
     - Simple factory, no state management (keeps it testable and composable)

  3. **sendMessage(socket: PartySocket, message: ClientMessage) function:**
     - `socket.send(JSON.stringify(message))`
     - Type-safe — only accepts ClientMessage union

  4. **parseServerMessage(data: string): ServerMessage | null function:**
     - `JSON.parse(data)` wrapped in try/catch
     - Validates `type` field exists (basic sanity check)
     - Returns typed ServerMessage or null on parse failure
     - Caller uses discriminated union switch on `type`

  5. **Do NOT create:**
     - React hooks (Phase 15)
     - Zustand integration (Phase 16)
     - Auto-reconnect logic beyond PartySocket's built-in
     - Connection state management (Phase 15 will own this in React state)

  Keep this file minimal — it's the bridge layer. Import `PartySocket` from `partysocket`, import types from `../types/protocol`.
  </action>
