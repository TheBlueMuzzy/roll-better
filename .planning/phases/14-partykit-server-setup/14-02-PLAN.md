**From 14-01:** Partykit scaffolded, protocol types defined. Server file has placeholder.

**Key design decisions:**
- Host-authoritative: server validates all actions, clients are dumb renderers
- First player to connect becomes host
- If host disconnects, next player becomes host (host migration)
- Room closes when last player leaves
- Max 8 players per room (matches game's 2-8 player constraint)
- Player IDs come from Partykit connection IDs (unique per connection)
- Room state broadcast after every player join/leave (full snapshot for simplicity)

**Partykit server patterns (from Context7):**
- `this.room.id` — the room identifier (will be the 4-letter code in Phase 15)
- `this.room.broadcast(message, [excludeIds])` — send to all except listed
- `this.room.getConnections()` — iterate all connections
- `conn.send(message)` — send to specific connection
- `conn.id` — unique connection identifier
- Server class is instantiated per room — state is isolated per room automatically

     - `hostId: string | null` — connection ID of the host
     - `status: RoomStatus` — "waiting" initially

  2. **onConnect handler:**
     - Send `connected` message with roomId and playerId (connection.id)
     - Do NOT add to players yet — wait for `join` message with name/color
     - If room is "closed" or at 8 players, send `error` and close connection

  3. **onMessage handler:**
     - Parse JSON, validate `type` field against ClientMessage union
     - Handle `join`:
       - Validate name is non-empty string (trim + check)
       - Create RoomPlayer: `{ id: conn.id, name, color, isHost: false, isReady: false }`
       - If first player (players.size === 0), set as host (isHost: true, hostId = conn.id)
       - Add to players map
       - Broadcast `player_joined` to others
       - Send full `room_state` to the joining player
     - Handle `leave`:
       - Remove from players (same as onClose logic — extract to helper)
     - Ignore unknown message types (log warning, don't crash)

  4. **Helper: removePlayer(connectionId)**
     - Remove from players map
     - If was host AND other players remain: assign new host (first remaining player)
     - If was host AND no players remain: set status to "closed"
     - Broadcast `player_left` to remaining players
     - If new host assigned, broadcast fresh `room_state` (host changed)

  Import types from `../src/types/protocol` (relative path from party/ to src/).
  Use `JSON.stringify` for all outgoing messages (Partykit sends strings over WebSocket).
  Wrap onMessage JSON.parse in try/catch — malformed messages get `error` response, not server crash.
