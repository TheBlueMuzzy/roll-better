<task type="auto">
  <name>Task 1: Apply MeshPhysicalMaterial with clearcoat to die body and pips</name>
  <files>src/components/Die3D.tsx</files>
  <action>
Replace the die body's meshStandardMaterial with meshPhysicalMaterial for premium quality:

Die body material:
- clearcoat: 1.0 (full glossy clearcoat layer)
- clearcoatRoughness: 0.1 (very smooth gloss)
- metalness: 0 (plastic, not metal)
- roughness: 0.35 (slightly smooth plastic)
- color: "#e8e0d4" (cream/ivory default — will become prop-driven)
- envMapIntensity: 1.0 (full HDRI reflections)

The HDRI environment map from drei's <Environment preset="apartment"> in Scene.tsx automatically applies to all meshPhysicalMaterial meshes — no manual envMap loading needed.

Also upgrade the pip material from meshStandardMaterial to meshPhysicalMaterial:
- clearcoat: 0.8
- clearcoatRoughness: 0.15
- metalness: 0
- roughness: 0.4
- color: "#ffffff" (white)

Both materials should remain module-level constants (not recreated per render). Since meshPhysicalMaterial is a Three.js class, create them with `new THREE.MeshPhysicalMaterial({...})` at module level, or use JSX `<meshPhysicalMaterial>` with useMemo to avoid re-creation.

Do NOT change geometry or pip positions from 02-01.
Do NOT add normal maps or texture maps yet — clean clearcoat is the target.
  </action>
  <verify>npm run build succeeds. Dev server shows die with visible glossy clearcoat reflections and specular highlights from HDRI environment.</verify>
  <done>Die body and pips use MeshPhysicalMaterial with clearcoat. HDRI reflections visible on die surface. No build errors.</done>
</task>

<task type="auto">
  <name>Task 2: Add player color tinting prop</name>
  <files>src/components/Die3D.tsx, src/components/Scene.tsx</files>
  <action>
Add a `color` prop to Die3D component:
- Type: `color?: string`
- Default: "#e8e0d4" (cream/ivory)
- Sets the die body meshPhysicalMaterial color
- Pip color stays white regardless of die body color

When the color prop changes, update the body material's color. Since the material is shared at module level, either:
(a) Clone the material per component instance and update its color, or
(b) Use JSX meshPhysicalMaterial (not module-level) so React handles prop updates

Option (b) is simpler for now with a single die. Use `<meshPhysicalMaterial>` as a JSX child of RoundedBox with the clearcoat props, and pass the color prop directly. Keep the pip material as a module-level constant (pips are always white).

Define player color constants in Die3D.tsx:
```
export const PLAYER_COLORS = {
  red: '#c0392b',
  blue: '#2980b9',
  green: '#27ae60',
  purple: '#8e44ad',
  orange: '#d35400',
  yellow: '#f39c12',
  teal: '#16a085',
  pink: '#e84393',
} as const;
```

In Scene.tsx, pass a color to Die3D to demonstrate (e.g., `color="#c0392b"` for red). This is temporary — game state will drive color in Phase 5.

Do NOT wire to Zustand store yet.
  </action>
