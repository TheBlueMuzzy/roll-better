<task type="auto">
  <name>Task 1: Fix touch target sizing and responsive tokens for landscape</name>
  <files>src/App.css</files>
  <action>
The `--touch-target` token uses `10vh` which resolves to ~18px on a landscape phone (180px viewport height at 2x). This is too small for finger taps.

**Fix --touch-target:** Change from `clamp(40px, 10vh, 56px)` to `clamp(36px, 10vmin, 52px)`. Using `vmin` picks the smaller of width/height — in landscape that's the height, giving roughly the same result as vh but with a more reasonable minimum. The key fix is the lower min (36px) still being finger-tappable while fitting the tighter landscape viewport.

Alternative if vmin doesn't feel right: `clamp(36px, 8vw, 52px)` — scale with long axis but clamped.

**Test:** After changing, verify that `.settings-gear`, `.tip-close`, and `.settings-close` buttons are all at least 36px in a 844x390 landscape viewport.
  </action>
  <verify>npm run build passes. Inspect --touch-target computed value in DevTools at 844x390 — should be 36-52px range, not sub-20px.</verify>
  <done>Touch targets render at finger-tappable size (36px+) in landscape viewport.</done>
