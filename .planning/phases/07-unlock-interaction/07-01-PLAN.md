<task type="auto">
  <name>Task 2: Settings items — Audio, Performance, Tips, Quit</name>
  <files>src/store/gameStore.ts, src/types/game.ts, src/components/Settings.tsx, src/App.tsx, src/App.css</files>
  <action>
Add four settings items to the panel:

1. **Settings state in Zustand**: Add a `settings` object to GameState:
   ```
   settings: {
     audioVolume: number;         // 0-100, default 80
     performanceMode: 'advanced' | 'simple';  // default 'advanced'
     tipsEnabled: boolean;        // default true
   }
   ```
   Add actions: `setAudioVolume(volume: number)`, `setPerformanceMode(mode)`, `setTipsEnabled(enabled: boolean)`.

   **Critical**: Do NOT reset `settings` in `initGame` or `initRound`. Settings persist across games. Add `settings` to `initialState` with defaults but do NOT spread it in reset actions. In `initGame`, only reset game-specific fields (players, phase, currentRound, roundState) — preserve settings.

2. **Audio Volume slider**:
   - Label "Audio" (left) + value display "80" (right) on same row
   - HTML range input: min=0, max=100, step=1
   - Styled: custom track (4px height, rgba(255,255,255,0.2) bg, white filled portion) and thumb (16px white circle)
   - Calls `setAudioVolume` on change
   - Note: no audio system exists yet (Phase 12) — this just stores the value

3. **Performance toggle**:
   - Label "Performance" + segmented control with "Simple" / "Advanced" options
   - Segmented control: two buttons side by side, active one has white background + dark text, inactive has transparent bg + white text
   - Calls `setPerformanceMode` on click
   - Note: no performance system exists yet — this just stores the value

4. **Tips toggle**:
   - Label "Tips" + on/off toggle switch
   - Toggle: 44x24px track, 20px circle thumb, slides left/right
   - Track color: off = rgba(255,255,255,0.2), on = #2ecc71 (green)
   - Calls `setTipsEnabled` on click

5. **Quit Game button**:
   - Full-width button at bottom of settings panel, separated by a subtle divider line
   - Style: border 1px solid rgba(255,255,255,0.2), transparent bg, white text, border-radius 8px, padding 12px
   - On click: show inline confirmation (replace button text with "Are you sure?" + Yes/No buttons)
   - On Yes: close settings panel, call `initGame(2)` + `initRound()` (restarts game)
   - On No: revert to normal Quit button

6. **Layout**: Stack settings items vertically with 20px gap. Each item row: flex, space-between, align-center. Add `.settings-item`, `.settings-slider`, `.settings-toggle`, `.settings-segmented`, `.settings-quit` CSS classes.

**What to avoid**: Don't use third-party UI libraries — pure HTML/CSS for all controls. Don't put the confirmation in a separate modal — inline it within the quit button area. Don't reset settings when quit/restart fires.
  </action>
