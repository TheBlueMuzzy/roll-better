Build the tip display and tracking system:

1. **Tip tracking in store**: Add `shownTips: string[]` to GameState (array of tip IDs that have been shown this session). Add action `showTip(tipId: string)` that appends to shownTips if not already present. Reset `shownTips` to `[]` in `initGame` (fresh session = fresh tips). Do NOT reset in `initRound` (tips persist within a game session).

2. **Tip helper function**: Create a utility function `shouldShowTip(tipId: string): boolean` that checks:
   - `settings.tipsEnabled` is true
   - `tipId` is NOT in `shownTips`
   Returns true only if both conditions met. Export from gameStore or a utils file.

3. **TipBanner component** (`src/components/TipBanner.tsx`):
   - Props: `text: string`, `onDismiss: () => void`
   - Renders a floating banner at the top of the screen (below the HUD top bar)
   - Position: absolute, top: 44px (below round/score bar), left/right: 16px, centered
   - Style: background rgba(0,0,0,0.75), backdrop-filter blur(8px), border-radius 10px, padding 10px 16px
   - Text: 13px, white, text-align center
   - Small X button on right side (optional — tapping anywhere on the banner also dismisses)
   - **Entry animation**: slides down from top (translateY: -20px → 0) + fades in (opacity 0 → 1) over 300ms
   - **Exit animation**: fades out (opacity 1 → 0) over 200ms, then calls onDismiss after transition
   - **Auto-dismiss**: starts a 4-second timer on mount. After 4s, triggers exit animation. Timer resets if user taps to dismiss early.
   - Use CSS transitions for animations (add/remove a `.tip-visible` class)
   - z-index: 20 (above HUD content at 10, below settings at 50)

4. **Tip state in App.tsx**:
   - `activeTip: { id: string; text: string } | null` as local React state
   - Helper: `tryShowTip(id: string, text: string)` — checks shouldShowTip, if true sets activeTip and calls showTip(id) in store
   - When activeTip is set, render `<TipBanner text={activeTip.text} onDismiss={() => setActiveTip(null)} />`
   - Only one tip at a time — if a tip is already showing, don't queue another (drop it)

5. **CSS classes**: `.tip-banner`, `.tip-banner.tip-visible`, `.tip-banner .tip-close`

**What to avoid**: Don't use react-transition-group or framer-motion — pure CSS transitions are sufficient. Don't show multiple tips simultaneously (one at a time, first-come-first-served). Don't persist shownTips to localStorage — session-only. Don't show tips during settings panel open.

<task type="auto">
  <name>Task 2: Initial game tips — roll, lock, unlock, must-unlock</name>
  <files>src/App.tsx</files>
  <action>
Wire up contextual tips that trigger at key gameplay moments:

1. **Tip: first-roll** — "Tap anywhere to roll your dice"
   - Trigger: when phase transitions to 'idle' AND currentRound === 1 AND roundState.rollNumber === 0
   - This is the very first idle of the game — player needs to know how to start
   - Show immediately on idle phase

2. **Tip: first-lock** — "Matched! Dice lock to the Goal row automatically"
   - Trigger: when phase transitions to 'locking' AND shownTips doesn't include 'first-lock' AND lastLockCount > 0
   - First time the player sees dice locking — explain what just happened
   - Show at start of locking phase

3. **Tip: first-unlock** — "Tap locked dice to select, then press UNLOCK"
   - Trigger: when phase transitions to 'unlocking' AND shownTips doesn't include 'first-unlock'
   - First time player enters unlock phase — they need to know the mechanic
   - Show at start of unlocking phase

4. **Tip: must-unlock** — "No dice to roll — you must unlock at least one"
   - Trigger: when phase transitions to 'unlocking' AND player.poolSize === 0 AND player.lockedDice.length < 8 AND shownTips doesn't include 'must-unlock'
   - Special case: explains why they MUST unlock (not optional)
   - Only shows if first-unlock tip has already been shown (don't stack explanations)
   - If first-unlock hasn't shown yet, show first-unlock instead (it's more fundamental)

5. **Implementation**: Add a useEffect in App.tsx that watches `phase` and triggers tips:
   ```
   useEffect(() => {
     if (phase === 'idle') tryShowTip('first-roll', '...');
     if (phase === 'locking' && lastLockCount > 0) tryShowTip('first-lock', '...');
     if (phase === 'unlocking') {
       if (mustUnlock && shownTips.includes('first-unlock')) {
         tryShowTip('must-unlock', '...');
       } else {
         tryShowTip('first-unlock', '...');
       }
     }
   }, [phase]);
   ```

6. **Tip timing**: Tips should not interfere with gameplay. The 4s auto-dismiss means tips disappear before most phase transitions complete. If a phase transition happens while a tip is showing, the tip stays (doesn't get yanked) — it will auto-dismiss on its own timer.

**What to avoid**: Don't show tips during the lobby/init phase — wait for the first real idle. Don't show more than one tip per phase transition (the tryShowTip guard handles this). Don't show the must-unlock tip before the player has seen the basic unlock tip. Don't add tips for scoring/handicap/roundEnd — those phases are self-explanatory from the HUD text, and too many tips cause fatigue.
  </action>

Turn tips off in settings → no tips appear.</verify>
  <done>4 contextual tips trigger at correct game moments: first-roll, first-lock, first-unlock, must-unlock. Each shows once per game session. Respects tips on/off setting. No tip stacking. Tips don't interfere with gameplay flow.</done>
