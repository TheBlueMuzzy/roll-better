<task type="auto">
  <name>Task 1: Carousel component with swipe + breadcrumb dots</name>
  <files>src/components/HowToPlay.tsx, src/App.css</files>
  <action>
Create a reusable carousel modal component:

1. **Component structure** (`src/components/HowToPlay.tsx`):
   - Props: `onClose: () => void`, `slides: Array<{ title: string; text: string; icon: string }>`
   - State: `currentSlide` (number, starts at 0)
   - Full-screen backdrop (same pattern as Settings: rgba(0,0,0,0.7), z-index: 60 — above settings)

2. **Carousel container**:
   - Centered card: max-width 340px, 92% width, background #1a1a1a, border-radius 16px
   - Overflow hidden on the slide track area
   - X close button: top-right, 32x32, absolute positioned, z-index above slides

3. **Slide track**:
   - Inner div with `display: flex`, all slides side-by-side
   - Transform: `translateX(-${currentSlide * 100}%)` with `transition: transform 0.3s ease-out`
   - Each slide is `flex-shrink: 0; width: 100%`

4. **Touch swipe**:
   - Track touch events on the slide container:
     - `onTouchStart`: capture `e.touches[0].clientX` in a ref
     - `onTouchMove`: track current X, optionally show drag preview (translate follows finger with resistance)
     - `onTouchEnd`: calculate delta X. If |delta| > 50px: advance/retreat slide. Otherwise snap back.
   - Clamp: can't swipe before slide 0 or after last slide
   - Use refs for touch tracking (not React state — avoid re-renders during drag)

5. **Mouse/desktop support**:
   - Left/right arrow keys advance/retreat (add keydown listener on mount, remove on unmount)
   - Optional: left/right click zones (10% edges of carousel) — subtle, not required

6. **Breadcrumb dots**:
   - Row of dots below the slide area, centered
   - Each dot: 8px circle, inactive = rgba(255,255,255,0.3), active = white
   - Dots are clickable — tap a dot to jump to that slide
   - Smooth active dot transition (scale 1.0 → 1.3 for active)

7. **CSS classes**: `.h2p-backdrop`, `.h2p-card`, `.h2p-close`, `.h2p-track`, `.h2p-slide`, `.h2p-dots`, `.h2p-dot`, `.h2p-dot--active`

**What to avoid**: Don't use a third-party carousel library (Swiper, etc.) — this is simple enough for pure React. Don't use CSS scroll-snap (it fights with controlled slide positioning). Don't put the swipe detection in React state — use refs for the touch coordinates, only setState for the final slide index. Don't allow momentum/overscroll past first/last slide.
  </action>

  <done>Carousel component with touch swipe, breadcrumb dots, keyboard arrows, smooth transitions. Reusable for any slide content.</done>
</task>
