**App.tsx:**

1. The existing `onGameStart` callback in App.tsx handles game initialization (initGame, initRound, etc.). For mid-game joins, the flow is slightly different because the game is ALREADY in progress — the player is joining mid-round with existing state.

   Check the existing onGameStart handler. It likely calls initGame + initRound. For a mid-game join (where rejoin_state contains the full state), the useOnlineGame hook's rejoin_state handler should sync the state directly using existing sync methods (syncAllPlayerState, etc.).

   The key insight: after the mid-game joiner's rejoin_state triggers gameStartData, `onGameStart` fires which calls `initGame`. Then `useOnlineGame` processes the rejoin_state and syncs the full state on top. This is the same flow as a reconnect — it already works.

   Verify this flow doesn't need changes by tracing:
   - gameStartData set → MainMenu effect fires onGameStart → App.tsx initGame called
   - useOnlineGame receives rejoin_state → calls existing sync logic
   - The order matters: initGame must happen before rejoin_state sync

   If there's a timing issue (rejoin_state arrives before onGameStart fires), add a guard: process rejoin_state sync after game is initialized.

2. For existing players (not the mid-game joiner): when they receive `seat_takeover`, the bot in their player list transitions to a human. This is handled by `seat_state_changed` (already in useOnlineGame). But the player's NAME and ID also change. The `seat_state_changed` message only carries playerId, seatState, seatIndex — not the new name.

   Solution: the phase_change that follows (or the next round_start) carries a full PlayerSyncState[] snapshot. The `syncAllPlayerState` method will update names, IDs, etc. from the snapshot. This is the existing reconciliation pattern — no changes needed.

3. Verify: the mid-game joiner should NOT trigger bot takeover logic in App.tsx (the useEffect watching localSeatState for 'bot'). Since the joiner starts as 'human-active' in the rejoin_state, this is fine.
