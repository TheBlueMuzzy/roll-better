**Server round transition flow (already implemented in party/server.ts):**
1. handleScoring sends `scoring` message (handled by 18-01)
2. After 2s delay: handleHandicapAndNextRound applies handicap to server state
3. If session over → sends `session_end` (handled by 18-01)
4. If not → sends `phase_change: roundEnd`, then after 500ms → serverInitRound sends `round_start`

**Local round transition flow (App.tsx useEffect, lines 225-278):**
1. phase === 'roundEnd': immediately start pool exit + goal exit
2. 500ms: initRound(skipPhase, goalValues), goal enter, pool spawn
3. 2000ms: goal settle, pool settle, setPhase('idle')

**Key insight:** Server sends roundEnd then round_start ~500ms apart — matching local timing exactly. Client handles roundEnd (start exits), then round_start (apply data + start enters).

**Constraining decisions:**
- initRound uses player.startingDice for poolSize — server handicap data must be applied BEFORE initRound
- initRound accepts {goalValues, skipPhase} options
- setPoolExiting, setGoalTransition, setPoolSpawning are existing store actions
- getSpawnPositions helper exists in App.tsx (line ~10) for pool spawn positions
