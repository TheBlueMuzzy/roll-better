**Existing AFK pattern (unlock timeout, party/server.ts lines 689-694):**
- checkWinnerOrUnlock starts 20s unlockTimeoutTimer
- autoSkipUnresponsivePlayers auto-skips players who haven't responded
- Must-unlock guard prevents skipping players with poolSize=0 who must unlock
- Timer is cleared when all responses are in (processAllUnlocks)

**Existing disconnect handling (party/server.ts removePlayer):**
- Marks player isOnline = false in gameState
- Removes pending unlock responses
- Re-checks unlockResponses if in unlocking phase
- Does NOT handle rolling phase — if a player disconnects during rolling, checkAllRolled will wait forever for their roll_result

**Constraining decisions:**
- Rolling AFK timer should use same pattern as unlock timeout (20s timer, auto-action)
- Timer starts when first roll_result arrives (phase transitions to rolling)
- On timeout: auto-roll random values for unresponsive players, process normally
- Disconnect during rolling: same behavior — treat disconnected player as auto-rolled
- cleanupTimers must include the new rolling timer
</context>

<task type="auto">
  <name>Task 1: Add rolling AFK timeout + disconnect-safe rolling</name>
  <files>party/server.ts</files>
  <action>
  **Part A — Rolling timeout timer:**
  1. Add a new timer field: `private rollingTimeoutTimer: ReturnType<typeof setTimeout> | null = null;`
  2. In `handleRollResult`, when transitioning from idle to rolling (first roll received), start the timer:
     ```
     if (this.gameState.phase === "idle") {
       this.gameState.phase = "rolling";
       // Start rolling AFK timeout — 20 seconds for all players to submit
       this.rollingTimeoutTimer = setTimeout(() => {
         this.rollingTimeoutTimer = null;
         this.autoRollUnresponsivePlayers();
       }, 20_000);
     }
     ```
  3. In `checkAllRolled`, clear the rolling timeout timer when all rolls are in (before processing bots):
     ```
     if (this.rollingTimeoutTimer) {
       clearTimeout(this.rollingTimeoutTimer);
       this.rollingTimeoutTimer = null;
     }
     ```
  4. Add `autoRollUnresponsivePlayers()` method:
     - Only runs if gameState exists and phase is "rolling"
     - For each online player who hasn't submitted (not in rollRequestedBy): generate random dice values (Array.from length poolSize, random 1-6), then process them exactly like handleRollResult would — compute locks via findAutoLocks, update server state, broadcast player_lock_result to all clients (broadcast to ALL, not broadcastExcept, since the AFK player didn't submit locally)
     - After processing all AFK players, call checkAllRolled() (which will process bots and transition to locking)
     - Log: `Auto-rolled for AFK player: ${player.name}`

  **Part B — Disconnect during rolling:**
  5. In `removePlayer`, add a rolling phase check (similar to existing unlocking check):
     ```
     // If we're waiting for roll results, remove this player's pending roll and re-check
     if (this.gameState.phase === "rolling" || this.gameState.phase === "idle") {
       // Player disconnected — they can't submit a roll
       // Don't add to rollRequestedBy — let autoRollUnresponsivePlayers or checkAllRolled handle it
       // But do check if all REMAINING online players have rolled
       this.checkAllRolled();
     }
     ```
     Since the player is now marked isOnline=false, checkAllRolled's filter `p.isOnline` will exclude them from the "all online players" check. This means if 3 players are online and one disconnects, checkAllRolled only waits for the remaining 2.

  **Part C — Timer cleanup:**
  6. Add rollingTimeoutTimer to cleanupTimers():
     ```
     if (this.rollingTimeoutTimer) {
       clearTimeout(this.rollingTimeoutTimer);
       this.rollingTimeoutTimer = null;
     }
     ```

  Do NOT modify the client — this is purely server-side.
  Do NOT change the unlock timeout — it already works at 20s.
  Use the same 20s timeout duration as unlocking for consistency.
