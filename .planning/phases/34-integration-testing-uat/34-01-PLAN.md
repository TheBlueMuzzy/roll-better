<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Disconnect/reconnect scenarios ready for testing. Tests the "phone call" and "tab close" PRD scenarios.</what-built>
  <how-to-verify>
    **Setup:** Open two browser windows (or PC + phone). Create a room, join from both, start a 2-player game.

    ### Scenario A: Phone call disconnect (reconnect within grace)
    1. During a **rolling phase**, close/disconnect one player's tab (simulate phone call)
    2. Observe: Other player should see the disconnected player's seat change to bot behavior
    3. Within the current phase timer, reopen the tab and rejoin the same room code
    4. Confirm: Player reclaims their seat, sees "You're back!" or reconnect toast
    5. Confirm: Game state is fully synced (correct score, correct round, correct phase)
    6. Confirm: AFK counter was reset (player isn't immediately at risk of bot promotion)

    ### Scenario B: Tab close — reconnect after grace expired
    1. During a **rolling phase**, close one player's tab entirely
    2. Wait for the phase to end AND the grace timer to expire (the disconnected player becomes a bot)
    3. Reopen a new tab, try to rejoin the room code
    4. Confirm: Player gets `room_closed` or is treated as a mid-game joiner (not auto-restored to old seat)
    5. If mid-game join: verify seat selection UI appears with available bot seats

    ### Scenario C: Reconnect during non-timed phase
    1. If possible, disconnect during a **locking** or **scoring** phase (non-timed)
    2. Confirm: Immediate bot promotion (no grace window for non-timed phases)
    3. Reconnect and verify behavior matches Scenario B (mid-game join path)

    Report which scenarios passed/failed and any unexpected behavior.
  </how-to-verify>
