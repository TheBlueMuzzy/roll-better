<task type="auto">
  <name>Task 2: Rework CSS design tokens for landscape viewport</name>
  <files>src/App.css</files>
  <action>
  **Problem:** All clamp() tokens use `vw` as the fluid value. In portrait, `vw` is the short axis (~390px on phones) so values like `3vw` = ~12px. In landscape, `vw` is the LONG axis (~844px) so `3vw` = ~25px — everything doubles in size.

  **Fix:** Convert fluid values from `vw` to `vh` for tokens that should scale with the short axis (which is now height in landscape). For layout tokens that should scale with width, keep `vw`.

  Replace the `:root` block (lines 43-63) with:

  ```css
  :root {
    /* Font sizes — scale with short axis (vh in landscape) */
    --fs-xs: clamp(10px, 2.5vh, 13px);
    --fs-sm: clamp(12px, 3vh, 14px);
    --fs-md: clamp(14px, 3.5vh, 18px);
    --fs-lg: clamp(16px, 4vh, 22px);
    --fs-xl: clamp(20px, 5vh, 28px);

    /* Spacing — scale with short axis */
    --sp-xs: clamp(4px, 1vh, 8px);
    --sp-sm: clamp(8px, 2vh, 14px);
    --sp-md: clamp(12px, 3vh, 20px);
    --sp-lg: clamp(16px, 4vh, 28px);
    --sp-xl: clamp(24px, 5vh, 36px);

    /* Layout — overlay/button widths scale with vw (long axis OK for width) */
    --overlay-max-w: clamp(280px, 45vw, 420px);
    --btn-max-w: clamp(240px, 40vw, 360px);
    --touch-target: clamp(40px, 10vh, 56px);
    --border-radius: clamp(8px, 2vh, 16px);
  }
  ```

  Key changes:
  - Font sizes + spacing: `vw` → `vh` (short axis governs text/spacing in landscape)
  - `--overlay-max-w`: `85vw` → `45vw` (85% of landscape width would be huge; 45% keeps overlays centered and readable)
  - `--btn-max-w`: `75vw` → `40vw` (same reasoning)
  - `--touch-target` + `--border-radius`: `vw` → `vh` (scale with short axis)
  </action>
  <verify>Run `npm run build` — no CSS errors. Grep for remaining `vw` in :root to confirm only layout width tokens use it.</verify>
  <done>All font/spacing/touch tokens use `vh`. Layout width tokens use `vw` with reduced percentages. No tokens produce oversized values in landscape.</done>
</task>
