<task type="auto">
  <name>Task 1: Create SoundManager with procedural dice sounds</name>
  <files>src/utils/soundManager.ts</files>
  <action>
Create a SoundManager module that:

1. **AudioContext lifecycle:**
   - Lazy-create AudioContext on first `init()` call
   - Export `initAudio()` — call on first user tap (autoplay policy)
   - Export `setVolume(v: number)` — takes 0-100 from store, maps to 0-1 gain
   - Master GainNode between all sounds and destination

2. **Procedural sound generators (keep simple):**
   - `playDiceImpact(force: number)` — short noise burst (20-40ms), bandpass filter ~800-2000Hz, volume scaled by force (clamp 0-1). Simulate wooden dice thud. Use a pre-generated noise buffer (create once, reuse).
   - `playDiceSettle()` — softer, lower-pitched thud (30ms noise burst, bandpass ~400-800Hz, volume 0.3). Plays when individual die stops.
   - `playAllSettled()` — short ascending two-tone chime (two sine oscillators, 600Hz then 800Hz, 80ms each, slight overlap). Signals "results are in."

3. **Noise buffer utility:**
   - Create a 1-second white noise AudioBuffer at init time
   - Reuse for all noise-based sounds (dice impacts)
   - Each play creates a new BufferSourceNode from the shared buffer

4. **Throttling:**
   - Dice impacts can fire rapidly (multiple dice bouncing). Throttle to max 8 concurrent impact sounds. If >8 playing, skip new ones.
   - Use a simple counter (increment on play, decrement on end).

Do NOT use Howler.js — pure Web Audio API. Keep total file under 150 lines. No classes — use module-level state (AudioContext, GainNode, noiseBuffer as let variables).
  </action>

Import SoundManager in a test component, call initAudio() on click, then playDiceImpact(0.5) — should hear a short thud sound. Check browser console for no errors.
  </verify>
