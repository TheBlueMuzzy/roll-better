<task type="auto">
  <name>Task 1: Create HUD overlay with roll button</name>
  <files>src/components/HUD.tsx, src/App.tsx, src/App.css</files>
  <action>
  **HUD component** (`src/components/HUD.tsx`):
  Create a React component (regular HTML, NOT inside Canvas) that overlays the game board with score display, round info, and a roll button.

  Props:
  - `score: number` — current player score
  - `targetScore: number` — session target (default 20)
  - `round: number` — current round number
  - `onRoll: () => void` — callback for the roll button
  - `isRolling: boolean` — true while dice are in flight (disables button)
  - `diceResults: number[] | null` — latest roll results to display

  **Layout (HTML/CSS, positioned over the canvas):**

  Top bar (position: absolute, top):
  - Round indicator: "Round 1" — left-aligned, small text (12px), muted white
  - Score: "0 / 20" — right-aligned, slightly larger (14px), white

  Bottom area (position: absolute, bottom):
  - Results display: when diceResults is not null, show "Results: 1, 2, 3, 4, 5" in a row above the roll button. Style: white text, 14px, text-shadow for legibility.
  - Roll button: large, centered, prominent. Text: "ROLL" (or "ROLLING..." when disabled).
    - Style: `background: rgba(255,255,255,0.15)`, `border: 2px solid rgba(255,255,255,0.3)`, `color: white`, `font-size: 18px`, `font-weight: bold`, `padding: 12px 48px`, `border-radius: 30px`, `cursor: pointer`, `backdrop-filter: blur(4px)`, `letter-spacing: 2px`
    - Disabled state: opacity 0.4, cursor: not-allowed
    - Position: centered horizontally, bottom: 24px (above build version)
    - `pointer-events: auto` on the button, `pointer-events: none` on the HUD container (so 3D clicks pass through everywhere except the button)

  **Styling approach:**
  - HUD container: position: absolute, inset: 0, pointer-events: none, z-index: 10
  - Individual interactive elements (button): pointer-events: auto
  - All text: font-family: system-ui, sans-serif; user-select: none

  Add CSS classes to `src/App.css`:
  - `.hud` — absolute overlay container
  - `.hud-top` — top bar flex row
  - `.hud-bottom` — bottom area, centered
  - `.roll-button` — styled roll button
  - `.roll-button:disabled` — dimmed state
  - `.hud-results` — results text above button

  **App.tsx changes:**
  - HUD needs access to the DicePool's rollAll(). Currently Scene handles this internally via refs.
  - Lift the roll trigger: Add `onRoll` callback from App to Scene. Scene exposes a `rollAll` function via forwardRef/useImperativeHandle on the Scene component. App calls it from HUD's onRoll.
  - Alternative (simpler): Use a Zustand action. Add `rollRequested: boolean` and `requestRoll: () => void` to gameStore. HUD calls `requestRoll()`, Scene reads `rollRequested` and triggers rollAll when it becomes true, then resets it.
  - Preferred approach: **forwardRef on Scene** — keeps it explicit and avoids store coupling.
    - Scene: wrap in forwardRef, expose `{ rollAll(): void }` via useImperativeHandle
    - App: hold sceneRef, pass `sceneRef.current?.rollAll` as onRoll to HUD
  - Also lift `diceResults` and `isRolling` state from Scene to App (or use a callback pattern):
    - Scene accepts `onResults: (results: number[]) => void` prop
    - App holds `diceResults` state and `isRolling` state
    - Scene calls `onResults` when dice settle
    - Scene calls an `onRollStart` callback when roll begins (to set isRolling true)
  - Remove the old `Html` result display from Scene (if not already removed in 04-01)
  - Keep floor click as a secondary roll trigger (still calls rollAll)
  - Render HUD as a sibling to Canvas in App.tsx (NOT inside Canvas)
  - Build version overlay stays below HUD (z-index 1000)
  </action>
  <verify>Run `npm run build` — no errors. Run dev server: clicking ROLL button triggers dice roll. Button shows "ROLLING..." during roll, returns to "ROLL" after settle. Results appear above button. Round and score display at top.</verify>

  <name>Task 2: Final layout spacing and proportion tuning</name>
  <files>src/components/Scene.tsx, src/App.css</files>
  <action>
  Review and tune the overall layout proportions for a polished game board feel.

  **Spacing check:**
  - GoalRow (Z ≈ -3.8): verify 8 dice clearly visible and not clipped by viewport edge. If too close to top edge, shift slightly toward center (e.g., Z = -3.5).
  - PlayerRow (Z ≈ -2.2): verify clear visual separation from both Goal row and rolling area. Adjust if too close to either.
  - Rolling zone back wall (Z = -0.8): verify dice have enough room to bounce without crowding the player row. If dice feel cramped, shift wall back slightly.
  - Roll button: verify it doesn't overlap with the rolling area visually. Bottom padding should keep it clear.

  **Visual polish:**
  - Add a subtle visual divider between the player row and the rolling area: a thin semi-transparent line mesh (planeGeometry, very narrow, stretched across the width) at Z ≈ -1.0, Y = 0.01, with low-opacity white or player-color material. This visually separates the "locked" zone from the "rolling" zone. Keep it subtle (opacity 0.1-0.15).
  - Alternatively, if a mesh divider looks odd from top-down, skip it and rely on the spatial gap alone. Use judgment based on visual result.

  **Responsive verification:**
  - The 9:16 CSS constrains the viewport. Verify the layout works at different window sizes by resizing the browser. The Canvas should scale proportionally and all zones should remain visible.
  - Ensure no elements are clipped on small screens (375px wide iPhone SE size).
