  <action>
  Create a PlayerRow component that renders 8 slot markers aligned 1:1 under the Goal row.

  Props:
  - `z?: number` — Z position of the row (default -2.2)
  - `color: string` — player color for slot outlines/dice
  - `lockedValues?: (number | null)[]` — array of 8 values (null = empty slot, number = locked die showing that face). Default: all nulls.

  **Slot markers (empty state):**
  Each slot is a thin, semi-transparent square outline on the floor at the slot position. Use a `<mesh>` with `<planeGeometry>` slightly smaller than DIE_SIZE (e.g., DIE_SIZE * 0.9) rotated flat (rotation [-PI/2, 0, 0]), with a `<meshBasicMaterial>` that is the player color at low opacity (0.15). This gives a subtle colored "ghost" square showing where locked dice will appear.

  Position each slot at:
  - X: `getSlotX(i)` (imported from GoalRow)
  - Y: 0.02 (just above floor to avoid z-fighting)
  - Z: the `z` prop

  **Locked die state:**
  When `lockedValues[i]` is not null, render a Die3D (scaled to DIE_SIZE) at that slot position instead of the ghost square. Use the player `color` prop. Apply `getRotationForFace(value)` (imported from GoalRow) so the correct face points up.

  Y position for locked dice: DIE_SIZE / 2 (resting on floor).

  Import SLOT_COUNT, getSlotX from GoalRow. Import DIE_SIZE from RollingArea. Import getRotationForFace from GoalRow. Import Die3D from Die3D.

  This component renders outside Physics (static visuals only, no RigidBody).
