**Layout conversion strategy:**

The camera is at Y=12 with 55° FOV looking down. For 16:9 landscape:
- Vertical visible range (Z): ~12.5 units (±6.25 from center)
- Horizontal visible range (X): ~22 units (±11 from center)

Current portrait layout stacks everything vertically (Z axis). For landscape:
- Keep same top-to-bottom conceptual layout (goal → players → rolling area)
- Widen arena significantly (more X room in landscape)
- Compress vertical spacing (less Z room)
- Goal slots can spread out more horizontally

**New layout constants (starting values — tune at checkpoint):**

| Constant | Portrait (old) | Landscape (new) | Rationale |
|----------|---------------|-----------------|-----------|
| ARENA_HALF_X | 2.8 | 3.8 | Wider for 16:9 (~35% increase) |
| ROLLING_Z_MIN | -1.7 | -0.5 | Pulled up — less depth needed |
| ROLLING_Z_MAX | 4.5 | 3.5 | Reduced depth — width compensates |
| DIE_SIZE | 0.589 | ~0.8 (7.6/9.5) | Larger dice in wider arena |
| Goal row Z | -4.67 | -3.2 | Closer — less vertical space |
| SLOT_SPACING | 0.62 | 0.84 | Wider — matches new DIE_SIZE |
| Goal indicators Z_OFFSET | -0.73 | -0.73 | Keep same relative offset |
| PROFILE_X_OFFSET | 0.10 | 0.10 | Keep same |

**Critical:** DIE_SIZE is derived from ARENA_HALF_X (`ARENA_HALF_X * 2 / 9.5`). Changing ARENA_HALF_X automatically changes DIE_SIZE. Everything downstream (slot spacing, positions, collider sizes) uses DIE_SIZE, so most things auto-scale. The main work is Z-axis repositioning.
