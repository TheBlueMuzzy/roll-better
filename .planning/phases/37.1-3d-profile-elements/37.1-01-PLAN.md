<objective>
Convert GoalProfileGroup from HTML overlay to 3D scene elements.

Purpose: HTML overlays (drei/Html) don't scale with the 3D camera — they maintain fixed pixel sizes regardless of viewport, causing overlap at small viewports and wasted space at large ones. Converting to 3D meshes + drei/Text makes profiles scale naturally with the scene like dice do.
Output: GoalProfileGroup renders as 3D elements (circle mesh + star sprite + Text score) that scale with the camera.
</objective>

**Current GoalProfileGroup renders:**
- White circle (48px scaled) with border
- Gold star emoji (★) at 62px scaled, centered over circle
- Score number inside star (16px bold white text)

**Target 3D equivalent:**
- CircleGeometry (MeshBasicMaterial white) for the background circle
- drei/Text for the star character (★) in gold
- drei/Text for the score number in white
- All as child meshes in a group, positioned in world space

**Key constraint:** R3F rule — never use React state for per-frame updates, mutate refs in useFrame.

  <action>
Replace the entire Html-based implementation with 3D elements:

1. **Remove** `Html` import from drei. **Add** `Text` import from `@react-three/drei` and `CircleGeometry`, `MeshBasicMaterial` from three (or use JSX equivalents).

2. **White circle background:**
   - `<mesh>` with `<circleGeometry args={[0.45, 32]} />` (radius ~0.45 world units ≈ roughly half a die)
   - `<meshBasicMaterial color="white" />`
   - Position at [0, 0, 0] within the group (the group's position prop handles placement)
   - Rotate to face camera: `rotation={[-Math.PI / 2, 0, 0]}` (since camera looks down from +Y)

3. **Gold star:**
   - `<Text>` from drei with text="★", fontSize ~0.7, color="#f1c40f"
   - Position slightly above the circle: y=0.01 (avoid z-fighting)
   - Same rotation as circle to face camera
   - anchorX="center", anchorY="middle"

4. **Score number:**
   - `<Text>` from drei with text={String(potentialScore)}, fontSize ~0.25, color="white", fontWeight bold (via font prop if needed, or just use default)
   - Position: y=0.02 (above star to avoid z-fighting)
   - Same rotation, centered anchors
   - Only render if potentialScore !== undefined

5. **Group wrapper:**
   - `<group position={position}>` wraps all elements
   - No Html, no CSS, no viewport-dependent scaling
   - The camera handles all scaling naturally

**What to avoid:**
- Do NOT use Html or any CSS. The whole point is pure 3D.
- Do NOT add responsive scaling logic (shortAxis / 390 etc.) — the camera handles this.
- Do NOT use React state for anything animated — use refs + useFrame if needed (currently GoalProfileGroup has no animations, so this is simple).
