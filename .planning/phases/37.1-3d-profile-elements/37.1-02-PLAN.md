   - Rotation: [-Math.PI / 2, 0, 0] to face camera

   - `<Text>` with text={name.charAt(0).toUpperCase()}, fontSize ~0.3, color="white"
   - Position: y=0.01 above avatar circle
   - Centered anchors

4. **Bot icon:**
   - If isBot, render `<Text>` with text="🤖", fontSize ~0.2
   - Position offset to top-left of avatar circle: x=-0.25, z=-0.25, y=0.02
   - If emoji doesn't render well in drei/Text, skip for now and log as enhancement

5. **Star + score (Column B top):**
   - Same pattern as GoalProfileGroup: star "★" in gold + score number in white
   - Position offset: x=0.55 from avatar center (to the right)
   - Star fontSize ~0.55, score fontSize ~0.2
   - Layered with y-offsets: circle at 0, star at 0.01, score at 0.02

6. **Stats text (Column B bottom):**
   - `<Text>` with text={`S${startingDice} | T${totalDice}`}, fontSize ~0.15
   - Color: rgba(255,255,255,0.7) — use "#ffffffb3" or similar
   - Position: x=0.55, z=0.25 (below star in screen space = +Z), y=0.01
   - Wrap in a group ref for the pop animation

7. **Handicap pop animation:**
   - Keep the useFrame logic but target a Three.js group ref instead of an HTML div ref
   - `const statsGroupRef = useRef<Group>(null)`
   - In useFrame: `statsGroupRef.current.scale.set(s, s, s)` instead of CSS transform
   - This is more R3F-idiomatic anyway

8. **Group wrapper:**
   - `<group position={position}>` wraps everything
   - No Html, no CSS, no viewport scaling math

**What to avoid:**
- Do NOT keep any Html, CSS, or viewport-dependent scaling
- Do NOT use React state for the pop animation — keep the existing useFrame + ref pattern
- Do NOT try to perfectly replicate pixel-level layout — approximate the spatial relationships in world units
