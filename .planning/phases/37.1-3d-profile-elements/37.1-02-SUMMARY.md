# Phase 37.1 Plan 02: PlayerProfileGroup 3D Conversion Summary

**PlayerProfileGroup converted to 3D with avatar circles, star shapes, stats text, and bot indicators — plus slot spacing increase and rendering fixes across both profile components**

## Performance

- **Duration:** 90 min (iterative with Muzzy feedback)
- **Started:** 2026-03-25
- **Completed:** 2026-03-25
- **Tasks:** 3 (2 auto + 1 checkpoint, with many iterative fixes)
- **Files modified:** 3

## Accomplishments
- PlayerProfileGroup: Html → 3D (avatar circle + letter Text + star ShapeGeometry + score Text + stats Text)
- Handicap pop animation converted from CSS transform to useFrame group.scale
- Stars flipped upright in both profile components
- Score text changed to black for readability
- Bot indicator: white dot with dark "B" letter, proper z-order layering
- SLOT_SPACING widened (1.05x → 1.2x) to fill left half better
- All materials: toneMapped=false to prevent ACES tone mapping washout
- Avatar circles: renderOrder=10 + depthWrite=false for reliable rendering above floor
- Bot dot: renderOrder=20 to draw above avatar

## Decisions Made
- Used plain "B" text for bot indicator (emoji not supported in drei/Text)
- renderOrder-based layering over y-offset alone (more reliable for flat overlapping meshes)
- Wider slot spacing rather than parent group scaling (preserves animation coordinate system)

## Deviations from Plan
Multiple iterative positioning/rendering fixes driven by Muzzy's live feedback. All were rendering corrections, not scope changes.

## Issues Encountered
- drei/Text doesn't render emoji (🤖) — default font lacks glyphs
- Three.js doesn't support 8-digit hex colors (#ffffffb3)
- ACES tone mapping washes out flat MeshBasicMaterial colors — fixed with toneMapped=false
- Depth buffer conflicts hide flat meshes near floor — fixed with renderOrder + depthWrite=false
