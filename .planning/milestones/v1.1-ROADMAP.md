# Milestone v1.1: Online Multiplayer

**Status:** SHIPPED 2026-03-05
**Phases:** 14-21
**Total Plans:** 20

## Overview

Take Roll Better from local AI play to real-time online multiplayer with friends, deployed and publicly playable.

**Key architectural decision:** Client-authoritative dice — each client rolls physics locally and reports settled values to server. Server computes locks via findAutoLocks and relays results to other clients. Per-player relay (no batching). Client-side buffered reveals (results hidden until you've acted, then animated).

**Constraints:** Free tier only (Partykit/Cloudflare 100k req/day). No accounts. 4-8 players. Host-authoritative validation.

## Milestone Summary

**Key Decisions:**
- Client-authoritative dice (each client rolls physics locally, reports values to server)
- Per-player relay with buffered reveals (no batching, results hidden until you've acted)
- sessionStorage for stable client ID (unique per tab, PartyKit requirement)
- 60s keepalive for reconnection window
- Snapshot + delta hybrid sync (full PlayerSyncState[] on every phase_change)
- AFK timers: 20s client-driven countdown, server-enforced fallback
- Static HTML for privacy policy (crawlable, no JS required)
- IARC 3+ rating (content warrants lower than conservative 13+)

**Issues Resolved:**
- Double-dice unlock bug (deferred snapshot, confirmUnlock guard)
- Locking stall (watchdog self-heal, server timeout race fix)
- Delta animation glitches (snapshot sync replaced fragile delta approach)
- Online roll pipeline rework (17-03 superseded by 17-04 for client-authoritative approach)

**Issues Deferred:**
- None outstanding

**Technical Debt Incurred:**
- Procedural audio stubs (Phase 13 stripped sounds, hooks retained for art pass)
- PWA icons are placeholders (pwa-192x192.png, pwa-512x512.png)
