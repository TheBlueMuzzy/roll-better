# Milestone v1.2: Polish

**Status:** SHIPPED 2026-03-06

## Overview

Clean up UI/UX rough edges — simplify menus, fix control issues, rework multiplayer screen flow. Remove unused features (shake-to-roll, manual difficulty selection), consolidate screens (merge LobbyScreen into MainMenu), and polish controls (gear icon, audio slider).

## Milestone Summary

**Key Decisions:**

- Shake-to-roll removed entirely (was unreliable, not core to experience)
- AI difficulty randomized per bot (removes unnecessary player choice)
- 4-state machine for inline online flow (idle/creating/joining/joined)
- menu-backdrop: position:absolute to stay within #root 9:16 bounds
- Slider fill via CSS --fill variable for proper thumb centering
- randomDifficulty() duplicated in server.ts (PartyKit bundles separately)
- HTP teaches mechanics only, not mode options
- ? icon bottom-left mirrors gear icon pattern

**Issues Resolved:**

- Audio slider had no visible fill indicator
- Gear icon positioned outside 9:16 play area (fixed with absolute positioning)
- Slider thumb not vertically aligned with track
- JOIN button remained active when creating a room (UAT fix: f858c41)

**Issues Deferred:**

- UAT-001: Room timeout issue (resolved itself — no longer reproduces)
- Drag-to-unlock interaction (deferred — see VISION.md)
- Hold-to-gather-roll (deferred — see VISION.md)

**Technical Debt Incurred:**

- randomDifficulty() duplicated between client and server (PartyKit bundle limitation)
